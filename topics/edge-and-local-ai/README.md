# 边缘端与本地化AI

> 不是所有场景都适合云端API——
> 网络不稳定、延迟要求高、隐私完全不能联网。
> 在手机、PC甚至离线环境中运行AI模型是真实需求。

---

## 核心技术

### 模型量化
> 📌 前置知识：[GPU计算](../../02-foundations/ai-infrastructure/gpu-computing/) 中的精度与量化部分。

把模型参数从FP16（2字节/参数）压缩到INT8（1字节）或INT4（0.5字节），
显著降低模型体积和推理所需的内存与算力。

### 知识蒸馏
用大模型（教师）的输出来训练小模型（学生），
让小模型在有限参数下尽可能接近大模型的能力。

### 本地推理引擎
- **llama.cpp**：C/C++实现，支持CPU和GPU推理，支持多种量化格式
- **Ollama**：基于llama.cpp的易用封装
- **ONNX Runtime**：跨平台推理引擎

---

## 适用场景

| 场景 | 推荐方案 |
|------|---------|
| PC本地运行 | Ollama + 量化模型（INT4/INT8） |
| 移动端 | ONNX Runtime + 小模型（1-3B参数） |
| 完全离线 | llama.cpp + 预下载的量化模型 |
| 浏览器内 | WebLLM（基于WebGPU） |

---

## 性能与质量的权衡

模型越小/量化越激进 → 推理越快、内存越少 → 但质量下降
模型越大/精度越高 → 质量越好 → 但需要更好的硬件

关键判断：在你的场景中，模型质量下降到什么程度是可接受的？
