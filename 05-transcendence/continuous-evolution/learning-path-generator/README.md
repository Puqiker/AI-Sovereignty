# 学习路径生成

---

## 根据当前状态生成学习路径

当你通过自我评估发现了需要提升的方向，
用以下方法为自己设计学习路径：

### 1. 明确目标

不是"我要学更多AI知识"（太模糊），
而是"我要理解RAG系统的架构，能够为公司的客服场景设计一个RAG方案"（具体可执行）。

### 2. 识别知识缺口

你已有的知识 → 目标所需的知识 → 缺口 = 需要学的内容

### 3. 寻找资源

优先顺序：
1. 本项目中的相关模块
2. 项目知识库中推荐的资源
3. 官方文档和权威教程
4. 社区讨论和实践经验
5. AI辅助学习（记住验证原则）

### 4. 设定时间框架

为每个学习目标设定明确的时间框架。
不是"有空就学"——那等于"永远不学"。

### 5. 实践验证

学完之后，通过实际项目来验证你真的理解了——
不是通过AI给你出的测试题（AI可能出错的题目），
而是通过你能否在实际场景中应用你学到的知识。

---

## AI辅助你的学习路径
你可以用AI来：
- 帮你评估知识缺口
- 帮你寻找和整理学习资源
- 帮你解答学习中的困惑（但记住验证）
- 帮你设计练习项目

你不应该让AI：
- 替你决定你应该学什么（这是你自己的判断）
- 成为你唯一的学习来源
- 替你做练习（你自己做才能真正理解）


---

## 关键认知

1. 学习路径应该基于具体的、可执行的目标——不是模糊的"学更多"
2. 识别知识缺口是设计路径的前提
3. 学完要通过实际项目验证——不是通过做题
4. AI辅助学习，但不替代学习
