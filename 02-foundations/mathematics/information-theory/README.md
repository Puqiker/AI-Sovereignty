## 信息量：一个事件的"意外程度"

### 从直觉出发

假设有两个新闻标题：
- "太阳明天照常升起"
- "明天地球将被小行星撞击"

哪条新闻包含的"信息量"更大？

显然是后者。因为前者是意料之中的事（概率接近1），
后者是极其意外的事（概率接近0）。

**信息量和概率成反比：越不可能发生的事件，包含的信息越多。**

数学上，一个概率为 p 的事件的信息量定义为：
I(p) = -log₂(p)

- 一个必然事件（p=1）：I(1) = -log₂(1) = 0（没有信息量）
- 一个概率0.5的事件：I(0.5) = -log₂(0.5) = 1（一个比特的信息）
- 一个概率0.01的事件：I(0.01) = -log₂(0.01) ≈ 6.64（很大的信息量）

### 它在AI中变成了什么

当模型预测下一个词时，如果它给真实答案的概率很高（比如0.9），
说明这个真实答案在模型看来是"意料之中"的，
对应的信息量 -log(0.9) 很小。

如果它给真实答案的概率很低（比如0.01），
说明真实答案对模型来说是"意外"的，
对应的信息量 -log(0.01) 很大。

**信息量越大 = 模型越"惊讶" = 模型预测得越差。**

## 熵：平均信息量

### 定义

对于一个离散概率分布 P，熵是信息量的期望值：
H(P) = -Σᵢ P(xᵢ) × log₂ P(xᵢ)

直觉上，熵衡量的是**一个分布的不确定性**：
- 如果分布非常集中（某个值的概率接近1），熵很低——你几乎确定会发生什么
- 如果分布很平坦（每个值的概率差不多），熵很高——你不确定会发生什么

### 它在AI中变成了什么

在语言模型中：

- 如果模型对下一个词非常确定（概率分布很尖锐），输出的熵很低
- 如果模型对下一个词很不确定（概率分布很平坦），输出的熵很高

一个好的模型，在应该确定的地方（"中华人民___"→"共和国"概率应该很高）应该输出低熵分布；
在确实有多种可能的地方（"今天我想吃___"→很多食物都合理）可以输出较高熵分布。

**模型的quality可以部分地通过它在不同上下文中输出的熵的合理性来判断。**

## 交叉熵：AI训练的核心损失函数

### 定义

给定两个分布——真实分布 P 和模型预测分布 Q，
交叉熵定义为：

H(P, Q) = -Σᵢ P(xᵢ) × log Q(xᵢ)

注意和熵的区别：
- 熵 H(P)：用分布P自己来编码P产生的信息，所需的平均信息量
- 交叉熵 H(P,Q)：用分布Q来编码P产生的信息，所需的平均信息量

交叉熵总是大于等于熵：H(P,Q) ≥ H(P)。
当且仅当 P = Q 时取等号。

### 它在AI中变成了什么

**交叉熵就是语言模型训练时使用的损失函数。**

在语言模型训练中：
- P 是真实分布（训练数据中下一个词的真实分布——实际上是one-hot的，即真实答案概率为1，其他为0）
- Q 是模型的预测分布（模型给出的概率分布）

当P是one-hot时，交叉熵简化为：
H(P, Q) = -log Q(真实的下一个词)

这就是你在上一模块看到的损失函数：**负对数似然（Negative Log-Likelihood）**。

你现在看到了从三个不同角度得到的同一个东西：
- 从概率论角度：最大似然估计 → 最大化 log P(真实|参数) → 最小化 -log P(真实|参数)
- 从信息论角度：最小化交叉熵 → 让模型分布Q尽可能接近真实分布P
- 从损失函数角度：让模型给真实答案分配尽可能高的概率

**三个视角，同一个数学表达式。**

这不是巧合——最大似然估计和最小化交叉熵在数学上是等价的。

### 为什么这很重要

当你看到AI论文或文档中提到"cross-entropy loss"时，
你现在知道它在说什么：让模型预测分布尽可能接近真实分布。

而且你知道它和最大似然估计的关系——它们是同一件事情的两种说法。

## KL散度：衡量两个分布的差异

### 定义

KL散度（Kullback-Leibler Divergence）衡量两个分布之间的"距离"：

D_KL(P || Q) = Σᵢ P(xᵢ) × log(P(xᵢ) / Q(xᵢ))

它和交叉熵的关系：
D_KL(P || Q) = H(P, Q) - H(P)

因为H(P)是一个常��（真实分布固定），
最小化交叉熵等价于最小化KL散度。

**KL散度 = 0 当且仅当 P 和 Q 完全相同。**

注意：KL散度不是对称的，D_KL(P||Q) ≠ D_KL(Q||P)。所以它不是严格意义上的"距离"，但在AI中经常被当作分布差异的度量。

### 它在AI中变成了什么

KL散度在AI中出现在很多关键的地方：

1. **训练目标的本质**：最小化模型和真实数据之间的KL散度

2. **RLHF中的约束**：当用人类反馈来调整模型时，会加一个KL散度惩罚项，防止模型偏离原始预训练分布太远。这就是那个公式：
目标 = 最大化人类偏好的奖励 - β × D_KL(微调后的模型 || 原始模型)
这里的KL散度确保模型在变得更"合人意"的同时不会丧失原有的语言能力。

3. **变分推断（VAE等）**：用KL散度来衡量近似后验和真实后验之间的差距。

4. **模型蒸馏**：教师模型的输出分布和学生模型的输出分布之间的KL散度，就是蒸馏的损失函数。

### 为什么这很重要

KL散度是AI中最常出现的信息论概念之一。
当你后面学到RLHF、模型蒸馏、变分自编码器时，
你会反复看到它。

如果你现在理解了它的直觉（衡量两个分布有多不同），
后面遇到它时就不会被公式吓住。

## 困惑度（Perplexity）：语言模型的常用评估指标

### 定义

困惑度是交叉熵的指数形式：
PPL = 2^(H(P, Q))

或者等价地：PPL = exp(-1/N × Σᵢ log Q(xᵢ))

其中 N 是序列长度，Q(xᵢ) 是模型给真实第 i 个词的概率。

### 直觉

困惑度可以理解为：**模型在每一步平均有多"困惑"。**

- 如果模型在每一步都非常确定正确答案（给真实词的概率接近1），困惑度接近1
- 如果模型在每一步完全随机猜（假设词表大小5万，每个词概率1/50000），困惑度等于50000
- 实际的好模型困惑度通常在个位数到几十之间

**困惑度越低，模型越好。**

### 为什么这很重要

当你看到AI模型的评测报告中提到"perplexity = 12.5"时，
你现在知道这意味着什么——模型在每一步平均在12.5个等可能的选项中做选择。
困惑度12.5比困惑度50好，因为模型更"确定"。

## 四个模块的整体图景

现在回顾一下你在数学基础部分学到的全部内容，
以及它们如何组合成AI的完整数学画面：

训练的完整数学过程：

数据被转换为向量（线性代数：嵌入）
↓
向量经过多层矩阵运算和非线性变换（线性代数：前向传播）
↓
最后一层输出经过softmax变成概率分布（概率论：softmax）
↓
预测分布和真实分布的差距用交叉熵衡量（信息论：损失函数）
↓
计算损失对每个参数的梯度（微积分：反向传播/链式法则）
↓
沿负梯度方向调整参数（优化：梯度下降）
↓
重复以上步骤直到损失足够小


**四块数学，拼出了AI训练的完整机制。**

你学过的线性代数给了计算骨架，
概率统计给了输出语义，
微积分给了学习能力，
信息论给了度量标尺。

没有哪一块是多余的，每一块都是必需的。

## 关键映射总结

| 信息论概念 | 在AI中的角色 |
|-----------|-------------|
| 信息量 -log(p) | 模型对真实答案的"意外程度" |
| 熵 H(P) | 分布的不确定性 |
| 交叉熵 H(P,Q) | 训练的损失函数——让模型分布逼近真实分布 |
| KL散度 D_KL(P\|\|Q) | 两个分布之间的差异——RLHF、蒸馏等的核心度量 |
| 困惑度 PPL | 语言模型的评估指标 |
