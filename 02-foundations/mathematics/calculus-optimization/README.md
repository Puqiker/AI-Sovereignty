# 微积分与优化 → AI

> 你学过导数、偏导数、链式法则、梯度。
> 这个模块告诉你一个简单到令人意外的事实：
> AI模型训练的核心算法——反向传播——本质上就是链式法则。
> 你在大学课堂上做过的求导练习题，就是训练神经网络的数学基础。

## 梯度：找到损失函数的下坡方向

### 你已经知道的

对于一个多元函数 f(x₁, x₂, ..., xₙ)，
梯度是各个偏导数组成的向量：

∇f = (∂f/∂x₁, ∂f/∂x₂, ..., ∂f/∂xₙ)

梯度指向函数值增长最快的方向。
反过来，**负梯度方向就是函数值下降最快的方向。**

### 它在AI中变成了什么

模型训练的过程可以这样理解：

1. 模型有一个**损失函数** L（衡量模型预测和真实答案之间的差距）
2. 损失函数的变量是模型的参数（几十亿个数字）
3. 训练的目标是找到使损失函数最小的参数值
4. 方法：沿着负梯度方向调整参数，让损失一步步减小

这就是**梯度下降（Gradient Descent）**：
θ_new = θ_old - η × ∇L(θ_old)

其中 η 是学习率（每一步走多大），∇L 是损失函数对参数的梯度。

直觉上的画面：
想象损失函数是一个高维的"地形"，有山峰有山谷。
模型的参数决定了你站在地形上的哪个位置。
梯度告诉你"从你当前位置往哪个方向走，地形下降最快"。
每一步沿着这个方向走一小步，逐渐走到山谷底部（损失最小的位置）。

### 为什么这很重要

**整个深度学习的训练过程，就是在做梯度下降。**

不管模型多大、架构多复杂、训练数据多庞大，
训练的核心循环都是：
1. 用当前参数计算预测结果
2. 计算预测结果和真实结果之间的损失
3. 计算损失对每个参数的梯度
4. 沿负梯度方向调整参数
5. 重复

变化的只是具体的损失函数、具体的梯度计算方法、以及学习率的调整策略。
但核心就是这四步。

## 链式法则：反向传播的全部

### 你已经知道的

链式法则说的是：如果 y = f(g(x))，那么：
dy/dx = (dy/dg) × (dg/dx)


也就是说，复合函数的导数等于各层导数的乘积。

更一般地，如果有一个计算链 x → g → f → h → L：

dL/dx = (dL/dh) × (dh/df) × (df/dg) × (dg/dx)


### 它在AI中变成了什么

神经网络就是一个巨大的复合函数。

输入 x 经过第一层变换得到 y₁，
y₁ 经过第二层变换得到 y₂，
y₂ 经过第三层变换得到 y₃，
……
最后一层的输出和真实标签比较得到损失 L。

x → [W₁, f₁] → y₁ → [W₂, f₂] → y₂ → [W₃, f₃] → y₃ → L


现在你要计算损失 L 对第一层参数 W₁ 的梯度（因为你需要这个梯度来调整 W₁）。

根据链式法则：∂L/∂W₁ = (∂L/∂y₃) × (∂y₃/∂y₂) × (∂y₂/∂y₁) × (∂y₁/∂W₁)


**这就是反向传播算法。**

"反向"的意思是：梯度从损失 L 开始，沿着计算链反方向传回去。

先算 ∂L/∂y₃（损失对最后一层输出的梯度），
然后乘以 ∂y₃/∂y₂（最后一层的局部梯度），得到 ∂L/∂y₂，
然后乘以 ∂y₂/∂y₁，得到 ∂L/∂y₁，
然后乘以 ∂y₁/∂W₁，得到 ∂L/∂W₁。

每一步都只需要计算当前层的局部导数，然后和后面传回来的梯度相乘。

**反向传播不是什么神秘的新算法。它就是链式法则在计算图上的系统化应用。**

### 为什么这很重要

很多人觉得反向传播是深度学习中最难理解的概念之一。
但如果你把它还原为链式法则，它就不难了——
它只是你在大学微积分课上做过的复合函数求导，
只不过函数的嵌套层数从两三层变成了几十上百层。

理解反向传播的好处：
1. 你知道为什么深度网络训练慢——层数越多，链式法则的乘积链越长，计算量越大
2. 你知道"梯度消失"是怎么回事——如果链中每个局部梯度都小于1，连乘之后总梯度趋近于零，前面的层就学不动了
3. 你知道"梯度爆炸"是怎么回事——如果局部梯度大于1，连乘之后总梯度指数增长，参数更新会变得不稳定
4. 你知道为什么ReLU激活函数流行——因为它在正区间的导数恒为1，避免了乘积链中的衰减

## 优化的现实：为什么不用解方程

### 你可能会想的问题

"既然目标是让损失函数最小化，
为什么不直接令梯度等于零，解方程求出最优参数？"

因为：

1. **参数太多**。几十亿个变量的方程组，没有解析解。

2. **损失函数不是凸的**。它有大量的局部最小值、鞍点、平坦区域。不像你在课本上见到的简单凸函数那样只有一个全局最小值。

3. **不需要全局最优**。实践表明，对于大型神经网络，一个"足够好的"局部最小值就能给出很好的性能。不需要找到绝对的全局最优。

所以训练用的是**迭代优化**——每一步沿梯度走一小步，
逐渐靠近一个足够好的参数区域。

### 梯度下降的变种

实际训练中用的不是最原始的梯度下降，而是一些改进版本：

- **随机梯度下降（SGD）**：不用全部训练数据算梯度（太慢），而是每次随机取一小批数据（mini-batch）来估计梯度。梯度估计有噪声，但速度快得多，而且噪声反而有助于跳出局部最小值。

- **Adam**：目前最常用的优化器。它在SGD的基础上引入了动量（参考之前的梯度方向，不容易因为噪声剧烈震荡）和自适应学习率（对不同参数使用不同的学习率）。

你不需要记住Adam的具体公式。你需要理解的是：
**所有这些优化器的核心思路都是梯度下降，区别只在于怎么利用梯度信息来决定步长和方向。**

## 损失函数的地形直觉

最后给你一个直觉性的画面来总结整个训练过程：

想象一个巨大的、几十亿维的地形。
地形的每一个点对应一组参数值。
地形的高度对应这组参数下的损失值。

训练开始时，你被随机扔到地形上的某个点（参数随机初始化）。
你站在那里，感受脚下的坡度（计算梯度）。
然后你沿着下坡方向走一步（参数更新）。
再感受坡度，再走一步。

你可能走进一个小坑（局部最小值），
也可能走到一个很平的区域（平坦区），
最终你会停在某个足够低的位置。

**训练完成后的模型参数，就是你最终停下来的那个位置。**

这个位置不是全局最低点——没有人知道全局最低点在哪。
但如果地形足够好（模型足够大、数据足够多），
你停下来的位置给出的模型性能就已经很好了。

## 关键映射总结

| 你学过的微积分概念 | 在AI中的角色 |
|------------------|-------------|
| 偏导数 | 损失函数对每个参数的变化率 |
| 梯度 | 告诉你参数应该往哪个方向调整 |
| 链式法则 | 反向传播算法的全部数学基础 |
| 梯度下降 | 模型训练的核心迭代过程 |
| 学习率 | 每一步走多大——太大震荡，太小太慢 |
| 局部最小值 | 训练可能停留的"不够好但还行"的位置 |
