# 概率统计 → AI

> 你学过概率、条件概率、贝叶斯公式、分布、期望、方差。
> 这个模块告诉你：AI从本质上就是一个概率系统。
> 它的输出是概率分布，它的训练目标是让概率分布和真实分布尽可能接近。
> 理解了这一点，你就理解了AI为什么会"幻觉"、为什么同一个问题可能给出不同回答、
> 以及为什么AI永远不是100%可靠的。

---

## 条件概率：AI在做的核心事情

### 你已经知道的

条件概率 P(A|B) 表示在事件B已经发生的条件下，事件A发生的概率。

P(A|B) = P(A ∩ B) / P(B)


### 它在AI中变成了什么

**大语言模型做的事情，可以用一句话概括：计算条件概率。**

当你输入"今天天气"这四个字，模型计算的是：

P(下一个词 | "今天天气")


也就是：在"今天天气"这个前文条件下，下一个词是什么的概率分布。

结果可能是：
- P("很好" | "今天天气") = 0.25
- P("不错" | "今天天气") = 0.18
- P("怎样" | "今天天气") = 0.15
- P("晴朗" | "今天天气") = 0.08
- ……

模型从这个概率分布中采样一个词，然后把这个词附加到输入后面，
再计算下一个词的条件概率，如此循环。

一整段话的生成过程，在数学上是：
P("今天天气很好适合出门") = P("今天") × P("天气"|"今天") × P("很好"|"今天天气") × P("适合"|"今天天气很好") × ...

这就是**自回归语言模型**的数学本质——逐步计算条件概率，然后逐步采样。

### 为什么这很重要

当你理解了这一点，你立刻就能明白几件事：

1. **为什么AI会"幻觉"**：模型计算的是"给定前文，下一个词最可能是什么"。如果前文把模型引导到了一个它训练数据中没有足够信息的方向，它仍然会输出一个概率最高的词——但这个词可能是事实上错误的。它不是在"说谎"，它是在做概率采样，而概率分布在这个区域不可靠。

2. **为什么同一个问题问两次可能得到不同回答**：因为模型不是在输出确定性答案，它是在从概率分布中采样。采样本身就有随机性。（这就是为什么有一个叫"temperature"的参数——它控制采样的随机程度。temperature越高，采样越随机；越低，越倾向于选概率最高的词。）

3. **为什么AI永远不是100%可靠的**：概率模型的本质就是"大多数时候对，但有一定概率错"。这不是bug，这是概率系统的固有特性。你不能指望消除它，只能管理它。

## 贝叶斯公式：机器学习的哲学基石

### 你已经知道的

P(A|B) = P(B|A) × P(A) / P(B)

或者更直观地写成：

后验概率 = 似然 × 先验 / 证据

你看到了新的证据B之后，你对A的信念从先验 P(A) 更新为后验 P(A|B)。

### 它在AI中变成了什么

**机器学习的本质，就是贝叶斯更新的大规模版本。**

模型有一组参数 θ（就是前面说的权重矩阵里的那几十亿个数字）。
训练之前，这些参数是随机初始化的——这是"先验"。

然后你给模型看大量的训练数据 D。
训练的过程，在贝叶斯的视角下，就是：

P(θ|D) ∝ P(D|θ) × P(θ)


用人话说：看到数据之后，模型对参数的信念从先验更新为后验。

实际操作中，大多数深度学习不是严格按照贝叶斯公式来计算的
（因为计算几十亿维参数的后验分布在数学上不可行），
而是用**最大似然估计（MLE）**来近似：
找到使 P(D|θ) 最大的参数 θ。

但贝叶斯的思想贯穿始终：
**模型通过观察数据来更新自己的"信念"（参数），使自己的预测越来越准。**

### 为什么这很重要

当你理解了训练的贝叶斯本质，你就能理解：

1. **为什么数据决定模型**：模型的能力不只取决于架构，更取决于它见过什么数据。垃圾数据训练出垃圾模型，这不是工程问题，这是统计学必然。

2. **为什么模型有偏见**：如果训练数据中某种观点出现的频率更高，模型的"后验信念"就会偏向那种观点。模型的偏见直接来自数据的偏见。

3. **为什么微调有效**：在一个已经训练好的模型（已经有了一个基于大量数据的后验）上继续用特定领域数据训练，就是在做进一步的贝叶斯更新——用新数据把模型的信念向特定领域调整。

## 概率分布：理解模型的输出

### 你已经知道的

概率分布描述了一个随机变量取各个值的概率。
离散分布用概率质量函数，连续分布用概率密度函数。
所有概率之和（或密度函数的积分）等于1。

### 它在AI中变成了什么

**模型的输出就是一个概率分布。**

对于语言模型来说，输出是一个离散分布——
在整个词表（可能有几万到几十万个token）上的概率分布：
P("的") = 0.12
P("了") = 0.08
P("很") = 0.06
P("是") = 0.05
...
（几万个token，概率之和 = 1）

这个分布是怎么从模型的数字输出变成概率的？

通过**softmax函数**：softmax(zᵢ) = exp(zᵢ) / Σⱼ exp(zⱼ)


模型最后一层输出的是一组原始分数（logits），
softmax把它们转换为一个概率分布——所有值变成正数，且加起来等于1。

softmax你在学概率的时候可能没见过，但它的数学很简单：
指数函数确保非负，除以总和确保归一化。
它的一个重要性质是：**它会放大差异**——
原始分数之间的小差距经过指数运算后会变成大差距，
使得概率分布更"尖锐"（最高分的选项获得更多概率质量）。

### 为什么这很重要

理解"模型输出是概率分布"这件事，让你能理解很多后续概念：

- **Temperature**：在softmax之前把logits除以一个温度参数T。T小则分布尖锐（模型更"确定"），T大则分布平坦（模型更"随机"）。

- **Top-p / Top-k采样**：不从完整分布中采样，而是只从概率最高的前k个token（或累计概率达到p的token集合）中采样。这是控制生成质量的手段。

- **困惑度（Perplexity）**：衡量模型预测能力的指标，本质上是模型输出分布和真实分布之间的差距。困惑度越低，模型预测越准。

## 期望与方差：理解模型的"平均表现"和"稳定性"

### 你已经知道的

期望 E[X] 是随机变量的"平均值"。
方差 Var(X) 衡量随机变量偏离期望的程度。

### 它在AI中变成了什么

- **期望**：当你评估一个模型的性能时，你不能只看一个测试用例。你需要在大量测试用例上计算性能指标的期望值。一个模型"准确率90%"的意思是：在测试数据的分布上，正确预测的期望概率是0.9。

- **方差**：两个模型可能有相同的平均性能，但一个方差很小（稳定地给出还行的结果），另一个方差很大（有时很好有时很差）。在生产环境中，低方差往往比高期望更重要——你宁可要一个稳定的80分，也不要一个在30分和100分之间跳动的模型。

- **偏差-方差权衡（Bias-Variance Tradeoff）**：这是机器学习中最核心的概念之一。一个模型的预测误差可以分解为偏差（系统性的错误倾向）和方差（对不同数据的敏感程度）。模型太简单→偏差高方差低（欠拟合）；模型太复杂→偏差低方差高（过拟合）。你在后面的机器学习模块会反复遇到这个概念。

## 最大似然估计：训练模型的数学目标

### 你已经知道的

给定一组观测数据 x₁, x₂, ..., xₙ，
最大似然估计（MLE）是找到使数据出现概率最大的参数值：

θ_MLE = argmax_θ P(x₁, x₂, ..., xₙ | θ)

通常取对数变成求和（因为连乘在数值计算上不稳定）：
θ_MLE = argmax_θ Σᵢ log P(xᵢ | θ)

### 它在AI中变成了什么

**语言模型的训练目标，就是最大似然估计。**

训练数据是大量的文本。每一句话就是一组观测数据。
模型的参数是权重矩阵中的数字。

训练的目标就是找到一组参数，使得模型在训练数据上给出的概率最大。

对于语言模型来说，最大化似然等价于最小化**交叉熵损失**
（这个会在信息论模块详细展开）：
Loss = -Σᵢ log P(真实的下一个词ᵢ | 前文ᵢ; θ)


用人话说：对于训练数据中每一个位置，
模型预测下一个词的概率分布中，
真实的下一个词应该获得尽可能高的概率。
如果模型给真实答案的概率很低（预测错了），损失就大。
训练就是不断调整参数来减小这个损失。

**整个训练过程，在数学上就是最大似然估计。**

### 为什么这很重要

这让你理解了训练的本质目标：
不是让模型"理解"语言，
而是让模型的概率分布尽可能拟合训练数据中的词汇分布。

模型"知道"的一切，都是通过这个概率拟合过程获得的。
它没有独立的知识存储——它的"知识"就是使得训练数据概率最大化的那组参数值。
