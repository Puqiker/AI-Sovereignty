# 数据基础

> AI基础设施模块的数据管线讲了数据"怎么处理"。
> 这个模块讲数据"怎么影响模型"——
> 数据质量的好坏、数据中的偏差、数据集的构建方式，
> 都会直接反映在模型的行为中。
> **理解数据和模型行为之间的因果关系，是判断AI系统可靠性的基础能力。**

---

## 核心观点

**你改变不了模型的架构（那是研究者的工作），
但你可以影响模型接触到什么数据——
而数据对模型行为的影响力不亚于架构。**

无论是预训练数据、微调数据，还是你放进上下文的参考文档，
数据的质量和特性都直接决定了模型输出的质量和特性。

## 完成标准

- [ ] 能列出评估数据质量的关键维度，并知道每个维度出问题时的后果
- [ ] 理解数据偏差的来源和传导机制——从数据到模型再到输出
- [ ] 能为微调任务设计数据集构建策略
- [ ] 面对模型的异常行为时，能判断是否是数据问题导致的

完成后，继续进入 [模型评估基础](../model-evaluation-foundations/)。
