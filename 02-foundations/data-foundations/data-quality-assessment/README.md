# 数据质量评估

> "垃圾进，垃圾出（Garbage In, Garbage Out）"不是一句口号，
> 它是AI系统中最硬的定律之一。
> 这个模块帮你建立评估数据质量的系统性框架。

---

## 数据质量的六个维度

### 1. 准确性（Accuracy）

数据反映的信息是否正确。

- 标注数据中的标签是否正确？一张猫的图片是否真的标注为"猫"？
- 文本数据中的事实性陈述是否准确？
- 数值数据是否在合理范围内？

**影响**：标注错误直接教给模型错误的模式。
一个标注错误率5%的数据集，模型的准确率理论上限就不会超过95%。

### 2. 完整性（Completeness）

数据是否覆盖了目标场景的各种情况。

- 训练数据是否包含了所有你需要模型处理的输入类型？
- 是否有某些类别的样本明显偏少？
- 是否有某些边界情况完全没有数据覆盖？

**影响**：数据中没有覆盖的场景，模型就不会在这些场景上有好的表现。
模型不会自己发明它从未见过的模式。

### 3. 一致性（Consistency）

数据在不同样本之间是否遵循相同的标准。

- 标注标准是否统一？不同标注员对"正面评价"的定义是否一致？
- 数据格式是否统一？日期是"2024-01-15"还是"01/15/2024"？
- 同一个事实在不同数据源中的描述是否一致？

**影响**：不一致��数据给模型发送矛盾的信号，
模型会在矛盾的模式之间"犹豫"，导致输出不稳定。

### 4. 时效性（Timeliness）

数据是否反映了当前的状态。

- 训练数据中的事实是否过时？
- 技术相关的内容是否还是最新的最佳实践？

**影响**：过时的数据导致模型给出过时的建议。
这在快速变化的领域（技术、法律、医学）中尤其危险。

### 5. 代表性（Representativeness）

数据的分布是否代表了真实世界的分布。

- 如果你的用户来自全球，训练数据是否只覆盖了英语？
- 如果你的应用面向所有年龄段，数据中是否某个年龄段显著过多？

**影响**：数据分布和真实分布不匹配，
模型在数据稀缺的群体或场景上表现会显著变差。

### 6. 无害性（Safety）

数据中是否包含有害内容。

- 是否有歧视性、攻击性的内容？
- 是否有隐私数据（个人信息、密码等）？
- 是否有会导致模型输出有害内容的训练样本？

**影响**：有害数据会让模型学到有害的模式，
增加模型在使用时产生有害输出的概率。

## 评估的实践方法

### 抽样审查

随机抽取一定比例的数据进行人工审查。
这是最直接也最可靠的质量评估方法。

建议：
- 至少抽查1%的数据（或至少几百条）
- 制定明确的检查清单
- 记录每种问题的出现频率

### 统计分析

- 数据量的分布：每个类别有多少样本？是否严重不均衡？
- 文本长度分布：是否有大量异常短或异常长的文本？
- 关键词分析：某些词或短语的出现频率是否异常？
- 重复率：有多少数据是完全相同或高度相似的？

### 模型辅助检测

用现有的模型来辅助检测数据质量问题：
- 语言模型计算困惑度——困惑度异常高的文本可能有质量问题
- 分类器检测有害内容
- 嵌入聚类发现异常样本

---

## 关键认知

1. 数据质量有六个维度——准确性、完整性、一致性、时效性、代表性、无害性
2. 每个维度的缺陷都会以特定方式影响模型行为
3. 抽样审查是最可靠的质量评估方法——没有捷径
4. 数据质量问题不会在训练中自动修复——它们会被模型忠实地学到
