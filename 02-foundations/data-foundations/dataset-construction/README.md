# 数据集构建

> 当你需要为微调或特定任务准备数据集时，
> 数据集的构建方式直接决定了微调后模型的能力和局限。
> 这不是一个简单的"收集数据"的过程，而是一个需要深思熟虑的设计过程。

---

## 微调数据集的设计原则

### 质量 > 数量

你在微调方法模块已经知道：
1000条高质量数据的微调效果可能好于10万条低质量数据。

**每一条训练数据都是在告诉模型"你应该这样做"。**
如果你给模型看了一条有错误的示范，
模型就学到了这个错误。

### 多样性

数据集应该覆盖你期望模型处理的各种场景。

如果你只给模型看"用列表格式回答知识问答"类型的数据，
模型就只擅长这一种场景。
当用户问一个需要详细论述的问题时，模型可能仍然用列表回答。

### 代表性

数据集的分布应该反映实际使用时的分布。

如果80%的用户问题是关于产品使用的，20%是关于价格的，
你的数据集大致也应该保持这个比例——
不要让某个类型的数据过多或过少。

### 难度梯度

数据集应该包含不同难度的样本：
- 简单的（模型容易做对的）
- 中等的（模型有时对有时错的）
- 困难的（模型经常做错的）

只有简单样本，模型学不到新东西。
只有困难样本，模型可能学不会。

## 标注方法论

### 制定标注规范

在开始标注之前，必须有明确的标注规范文档：
- 每个标签的精确定义
- 边界情况的处理指南
- 标注示例（正例和负例）
- 不确定情况的处理方式

### 标注质量控制

- **多人标注**：同一条数据由多个人标注，取一致的结果（或多数投票）
- **标注员一致性检查**：计算标注员之间的一致性（如Cohen's Kappa）
- **黄金数据检查**：在数据中混入已知正确答案的样本，检测标注员的准确率
- **定期校准**：标注员之间定期讨论边界案例，统一标准

### 合成数据

用AI生成训练数据（如用强模型生成弱模型的训练数据）：

优点：
- 快速获得大量数据
- 成本低于人工标注

风险：
- AI生成的数据可能包含AI的典型错误模式
- 用AI数据训练AI可能导致"模型坍塌"——模型只学到AI的风格而失去多样性
- 合成数据的质量需要严格验证

**合成数据适合作为人工标注数据的补充，不适合作为唯一来源。**

---

## 关键认知

1. 微调数据集是一个需要设计的产品——不是随便收集的
2. 质量 > 数量 > 多样性——三者排序
3. 标注规范是数据质量的基础——没有规范的标注等于随机标注
4. 合成数据是有用的补充，但不能完全替代人工标注
5. 数据集的局限性会直接成为微调后模型的局限性
