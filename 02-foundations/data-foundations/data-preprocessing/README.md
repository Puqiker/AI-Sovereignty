# 数据预处理

> 原始数据几乎不可能直接用于训练。
> 数据预处理就是把杂乱的原始数据变成干净、规范、可用的训练数据的过程。

---

## 文本数据的预处理

### 清洗

- 移除HTML标签、特殊字符、控制字符
- 统一编码格式（UTF-8）
- 修复乱码和编码错误
- 移除重复空白、换行符规范化

### 规范化

- 统一大小写（取决于任务需求——有些任务大小写有意义，有些没有）
- 统一标点符号（全角/半角）
- 数字格式统一
- URL和邮箱地址处理（移除或替换为占位符）

### 分段

- 长文本切分为合适的训练片段
- 切分时需要考虑语义完整性——不要在句子中间切断
- 不同的切分策略会影响模型学到的上下文模式

### 数据增强

在数据量不足时，通过变换已有数据来"创造"更多数据：

- 同义词替换："好的" → "不错的"
- 回译（Back Translation）：中→英→中，生成措辞不同但含义相同的文本
- 随机删除/交换词语（需谨慎，可能改变语义）

**注意**：数据增强不能创造真正的新信息，
它只是增加了已有信息的多样性。过度增强可能引入噪声。

## 结构化数据的预处理

### 缺失值处理

- 删除含缺失值的样本（数据充足时）
- 用均值/中位数/众数填充（简单但可能引入偏差）
- 用模型预测填充（更复杂但可能更准确）

### 特征缩放

不同特征的数值范围差异很大时，需要缩放到统一范围：
- 标准化（减均值除标准差，使均值0方差1）
- 归一化（缩放到[0,1]区间）

这对于使用梯度下降的模型很重要——
特征尺度不一致会导致某些参数的梯度过大或过小。

### 类别编码

将非数值的类别特征转换为数字：
- 独热编码（One-hot）：每个类别一个维度
- 标签编码（Label Encoding）：每个类别一个整数

---

## 预处理的原则

1. **可重复性**：预处理步骤应该是可记录、可重复的。使用脚本而非手动操作。

2. **保留原始数据**：永远保留一份未经处理的原始数据。预处理可能引入错误，你可能需要重新处理。

3. **先理解再处理**：在做任何预处理之前，先理解数据的特性。盲目应用预处理步骤可能适得其反。

4. **预处理和评估的一致性**：训练数据的预处理方式必须和推理时的处理方式一致。否则模型在训练和使用时"看到"的数据形态不同，性能会下降。

---

## 关键认知

1. 预处理是把杂乱原始数据变成可训练数据的必要环节
2. 文本预处理的核心：清洗→规范化→分段→（可选）数据增强
3. 预处理必须可重复、有记录——使用脚本而非手动操作
4. 训练和推理的预处理必须一致——否则会有性能落差
