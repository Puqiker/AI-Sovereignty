# CNN架构

> 全连接网络处理图像时把每个像素当作独立的输入，完全忽略了空间关系。
> 卷积神经网络（CNN）通过一个核心思想解决了这个问题：
> **局部连接 + 权重共享。**
> 这个思想极其优雅，理解它不仅帮你理解图像AI，
> 更帮你理解一个普遍的设计原则：利用数据的结构来简化模型。

---

## CNN为什么存在

先量化一下全连接网络处理图像的问题：

一张 224×224 的彩色图片有 224×224×3 = 150,528 个像素值。
如果第一个隐藏层有1000个神经元，
权重矩阵的大小是 1000×150528 = 1.5亿个参数。

这只是第一层。一个深度网络可能有几十层。
参数量爆炸，训练不可行。

更关键的是：这些参数中的绝大多数是浪费的。
因为图像有两个重要的结构性质：

### 性质一：局部性

图像中有用的特征是局部的——
一只眼睛、一条边缘、一个纹理——这些特征只占据图像的一小块区域。
一个神经元不需要"看"整张图片来检测一条边缘，
它只需要看一个小区域就够了。

### 性质二：平移不变性

同样的特征可能出现在图像的任何位置。
猫的耳朵可能在左上角、也可能在右下角。
你不应该为左上角的耳朵和右下角的耳朵训练两个不同的检测器——
同一个检测器应该在整张图上都能工作。

CNN正是基于这两个性质设计的。

## 卷积操作

### 基本思想

不是让一个神经元连接到所有输入像素，
而是让它只连接到一个小区域——比如 3×3 的一个方块。

这个 3×3 的小权重矩阵叫做**卷积核（kernel/filter）**。

这个卷积核在整张图上滑动——
每到一个位置，就和那个位置的 3×3 像素做加权求和，产生一个输出值。

输入图像（5×5）： 卷积核（3×3）： 输出（3×3）：
[1 2 3 4 5] [1 0 1] [? ? ?]
[6 7 8 9 0] [0 1 0] [? ? ?]
[1 2 3 4 5] [1 0 1] [? ? ?]
[6 7 8 9 0]
[1 2 3 4 5]

位置(0,0)的计算：
1×1 + 2×0 + 3×1 + 6×0 + 7×1 + 8×0 + 1×1 + 2×0 + 3×1 = 15


卷积核滑过整张图，产生一张**特征图（feature map）**。

### 这解决了什么问题

**局部连接**：每个输出位置只依赖于输入的一个小区域（3×3），
而不是整张图。参数量从 150万 × 1000 暴降到 3×3×3 = 27个参数（一个卷积核）。

**权重共享**：同一个卷积核在图像的所有位置共享。
左上角和右下角使用的是完全相同的权重。
这就实现了平移不变性——同样的模式不管出现在哪里，都能被同一个卷积核检测到。

### 多个卷积核

一个卷积核只能检测一种模式（比如水平边缘）。
实际中会使用多个卷积核（比如64个或128个），
每个学习检测不同的模式。

64个卷积核在一张输入图上运算，产生64张特征图。
这64张特征图作为下一层的输入。

## CNN的完整结构

一个典型的CNN由以下几种层组成：

### 卷积层（Convolutional Layer）

用多个卷积核提取特征。
输出 = 激活函数(卷积(输入, 卷积核) + 偏置)

### 池化层（Pooling Layer）

降低特征图的分辨率。
最常见的是最大池化（Max Pooling）：
在每个 2×2 的区域中取最大值，特征图尺寸缩小一半。

[1 3]
[5 2] → 最大池化 → [5]

池化的作用：
- 减少计算量和参数量
- 引入少量平移不变性（特征移动一两个像素不影响池化结果）
- 逐步扩大后续层的"感受野"（每个神经元间接看到的输入区域）

### 全连接层

在卷积和池化提取了足够的特征之后，
最后用全连接层做分类决策。

### 完整流程

输入图像
↓
[卷积层 → 激活 → 池化] × N层 ← 特征提取
↓
展平为一维向量
↓
[全连接层] × M层 ← 分类决策
↓
softmax → 类别概率


越靠前的卷积层学习越低级的特征（边缘、颜色渐变）。
越靠后的卷积层学习越高级的特征（对象部件、完整对象）。

---

## 感受野：每个神经元"看"到多大的区域

第一层的一个神经元只看到 3×3 的区域。
第二层的一个神经元基于第一层的特征图计算，
而第一层的每个位置已经看过了 3×3 的区域。
所以第二层的一个神经元间接看到了 5×5 的原始输入区域。

层数越多，每个神经元的感受野越大。

这就是深度CNN能理解图像中大范围模式的原因——
虽然每一层只做局部计算，但通过多层叠加，
后面的层能间接"看到"整张图像。

## CNN为什么不直接用于文本

CNN是为网格状数据（图像）设计的。
文本虽然也可以用CNN处理（1D卷积），
但文本有一个图像没有的挑战：**长距离依赖**。

在一个句子中，第一个词可能和最后一个词有关系：
"那个昨天在公园里遇到的穿红色外套的**女孩**是我的**同学**"

CNN的感受野是局部的，要捕捉这么远的依赖需要非常多层。
这就是为什么处理文本需要不同的架构——RNN和后来的Transformer。

---

## CNN的核心设计原则

CNN的成功不仅仅是技术成就，它体现了一个深刻的设计原则：

**当你知道数据有某种结构时，把这种结构信息编码到模型架构中，
模型就能用更少的参数、更少的数据、更快地学到更好的表示。**

- 图像有空间局部性 → 用局部卷积核
- 相同模式可能出现在不同位置 → 权重共享

这个原则在后面的所有架构中都会反复出现：
- RNN把"时间顺序"编码到了架构中
- Transformer把"任意位置之间都可能相关"编码到了架构中

---

## 关键认知

1. CNN的核心思想：局部连接 + 权重共享——利用图像的空间结构
2. 卷积核在整张图上滑动，用相同的权重检测相同的模式
3. 多层卷积 = 从低级特征到高级特征的层级抽象
4. 池化层降低分辨率、扩大感受野
5. CNN不擅长处理长距离依赖——文本处理需要不同的架构
6. 核心设计原则：把数据的结构信息编码到模型架构中
