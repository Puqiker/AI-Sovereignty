# 注意力机制

> RNN的瓶颈在于它必须把所有历史信息压缩到一个固定大小的向量中。
> 注意力机制的核心洞察是：
> **不要压缩，而是在需要的时候直接回去看原始信息。**
> 这个看似简单的想法，彻底改变了整个AI领域。

---

## 从一个翻译问题说起

注意力机制最初是在机器翻译中被提出的。

假设你要把一个英语句子翻译成中文：
英语：The cat sat on the mat
中文：猫 坐在 垫子 上

用RNN做翻译的传统方式：
编码器RNN：The → cat → sat → on → the → mat → [压缩成一个向量 h]
解码器RNN：[h] → 猫 → 坐在 → 垫子 → 上


所有英语句子的信息都被压缩到了一个向量 h 中。
解码器在生成每一个中文词时，都只能从这一个 h 中获取信息。

问题很明显：
- 当生成"猫"时，最相关的英语词是"cat"——但"cat"的信息已经和其他所有词混在 h 里了
- 当生成"垫子"时，最相关的英语词是"mat"——但解码器看到的还是同一个 h
- 句子越长，h 里的信息越稀释，翻译质量越差

---

## 注意力的核心思想

注意力机制的解决方案：

**在生成每一个输出词时，让解码器回去看编码器的每一个位置，
根据当前需求决定应该"关注"哪些位置。**

编码器RNN：The → cat → sat → on → the → mat
h₁ h₂ h₃ h₄ h₅ h₆

生成"猫"时：
回去看所有位置的隐藏状态 h₁...h₆
发现 h₂（"cat"）最相关 → 给 h₂ 最高的注意力权重
加权组合：context = 0.05×h₁ + 0.80×h₂ + 0.05×h₃ + 0.03×h₄ + 0.02×h₅ + 0.05×h₆
用这个 context 来帮助生成"猫"

生成"垫子"时：
再次回去看 h₁...h₆
发现 h₆（"mat"）最相关 → 给 h₆ 最高的注意力权重
用新的 context 来帮助生成"垫子"


**每生成一个词，注意力权重都重新计算。**
不同的输出位置关注不同的输入位置。

这就是"注意力"这个名字的来源——模型学会了在不同时刻"注意"不同的位置。

## 注意力的数学

你在数学基础已经有了理解注意力机制所需的全部数学。

### 注意力权重的计算

核心操作：**用当前位置的向量去和所有历史位置的向量计算相似度。**

相似度怎么算？**向量点积。**

得分 = 当前向量 · 历史位置向量


你在线性代数模块学过：两个向量的点积反映它们的相似度。
方向越接近，点积越大。

计算出和每个历史位置的得分之后，用 **softmax** 把它们转换成概率分布：
注意力权重 = softmax(所有得分)

softmax确保权重之和为1——这就是一个概率分布，
表示当前位置应该把多少"注意力"分配给每个历史位置。

### 上下文向量的计算

有了注意力权重后，对所有历史位置的向量做加权求和：context = Σᵢ αᵢ × hᵢ

其中 αᵢ 是位置 i 的注意力权重，hᵢ 是位置 i 的向量表示。

**注意力机制的全部操作就是：点积算相似度 → softmax归一化 → 加权求和。**

这三步操作的数学工具你全都学过了。

## Query, Key, Value：注意力的标准表述

在现代的注意力机制中，上面的过程被形式化为三个概念：

- **Query（查询）**：当前位置的向量——"我在找什么？"
- **Key（键）**：每个历史位置的标识向量——"我是谁？"
- **Value（值）**：每个历史位置的内容向量——"我包含什么信息？"

计算过程：
1. 用 Query 和每个 Key 做点积，得到相似度得分
2. 用 softmax 把得分转换成注意力权重
3. 用注意力权重对 Value 做加权求和，得到输出

用数学公式写就是那个著名的注意力公式：
Attention(Q, K, V) = softmax(Q × K^T / √d_k) × V

其中 √d_k 是一个缩放因子（d_k 是Key向量的维度），
用来防止点积值太大导致softmax的输出过于"尖锐"。

### 为什么分成 Query、Key、Value

一个自然的问题：为什么不直接用同一个向量做所有事情，
而要分成三个？

因为"我在找什么"、"我是谁"、"我包含什么信息"
是三个不同的角色。

类比：你在图书馆找书。
- Query 是你的搜索需求："我想找关于量子计算的书"
- Key 是每本书的标签/关键词："量子物理""计算机科学""量子计算入门"
- Value 是每本书的实际内容

你用搜索需求（Query）和书的标签（Key）匹配，
找到最相关的书，然后读取它们的内容（Value）。

如果搜索需求和书的标签使用同一种表示，搜索能力会受限。
分开表示让匹配过程更灵活。

## 注意力解决了RNN的两个问题

### 问题一：信息瓶颈 → 解决了

不再把所有信息压缩到一个固定向量。
需要什么信息，直接回去看对应位置的原始表示。
序列再长，每个位置的信息都被完整保留。

### 问题二：无法并行 → 部分解决

注意力的计算——所有位置的点积——可以用矩阵乘法一次性算完：
所有位置的注意力 = softmax(Q × K^T / √d_k) × V


Q、K、V 都是矩阵（所有位置的向量堆叠起来），
整个计算是矩阵乘法——**天然适合GPU并行**。

但如果注意力仍然嵌套在RNN中，整体仍然是顺序的。
要彻底解决并行问题，需要完全抛弃RNN——这就是Transformer做的事情。

## 自注意力（Self-Attention）

在翻译的例子中，注意力是编码器和解码器之间的——
解码器的当前位置"关注"编码器的各个位置。

**自注意力**更进一步：**一个序列中的每个位置关注同一序列中的所有其他位置。**

输入序列："今天天气很好"

"今天"关注所有位置：今天(自己), 天气, 很, 好
"天气"关注所有位置：今天, 天气(自己), 很, 好
"很"关注所有位置：今天, 天气, 很(自己), 好
"好"关注所有位置：今天, 天气, 很, 好(自己)


每个位置都从所有位置中收集相关信息，
根据内容的相关性决定应该关注哪些位置。

**自注意力是Transformer的核心操作。**

### 自注意力的计算

每个位置的输入向量 xᵢ 通过三个不同的线性变换产生 Q、K、V：
qᵢ = W_Q × xᵢ （查询向量）
kᵢ = W_K × xᵢ （键向量）
vᵢ = W_V × xᵢ （值向量）

W_Q、W_K、W_V 是三个可学习的权重矩阵。

然后按标准注意力公式计算。

### 为什么自注意力这么强大

自注意力让序列中任意两个位置之间可以直接交互——
不管它们相隔多远。

在RNN中，位置1的信息要传到位置100，需要经过99步的顺序传递，
信息在传递过程中不断稀释。

在自注意力中，位置1和位置100之间只有一步的距离——
直接做点积计算相似度。

**这彻底解决了长距离依赖问题。**

## 关键认知

1. 注意力的核心思想：不压缩历史信息，需要时直接回去看
2. 数学操作：点积算相似度 → softmax归一化 → 加权求和
3. Query/Key/Value 分离了"搜索需求""标识"和"内容"三个角色
4. 自注意力让同一序列中任意两个位置可以直接交互
5. 注意力解决了信息瓶颈问题，自注意力进一步解决了长距离依赖问题
6. 注意力的计算是矩阵乘法，天然适合GPU并行
