# Transformer架构

> 2017年，Google的论文"Attention Is All You Need"提出了Transformer架构。
> 它的核心主张就是标题所说的：**注意力就是你所需要的一切。**
> 不需要RNN的循环，不需要CNN的卷积，只用注意力机制就能处理序列数据。
> 这个架构成为了之后所有主流大模型——GPT、BERT、Claude、Llama——的基础。
>
> 这是筑基阶段最关键的一个模块。
> 理解了Transformer，你就理解了当前AI的物理结构。

---

## 为什么完全抛弃RNN

上一模块你学到了注意力机制的两大优势：
解决信息瓶颈和长距离依赖。

但如果注意力仍然嵌在RNN框架里，
整体的顺序计算限制仍然存在。

Transformer的关键决策是：**完全不用循环结构。**

整个模型的核心操作只有：
1. 自注意力（处理位置间的关系）
2. 前馈网络（处理每个位置自身的特征变换）
3. 残差连接 + 层归一化（稳定训练）

没有循环，没有递归。
所有位置的计算可以**完全并行**。

这使得Transformer可以在GPU上极其高效地训练，
从而能够扩展到几千亿参数的规模——这是RNN永远做不到的。

---

## Transformer的完整结构

### 原始的编码器-解码器结构

Transformer最初是为翻译任务设计的，有编码器和解码器两部分：
编码器（Encoder）：
输入序列 → [自注意力 → 前馈网络] × N层 → 编码表示

解码器（Decoder）：
已生成的输出 → [掩码自注意力 → 编码器-解码器注意力 → 前馈网络] × N层 → 下一个词的概率


### 当前大语言模型主要用解码器

GPT系列、Llama、Claude等大语言模型主要使用的是**只有解码器的结构**。

因为语言模型的任务是"给定前文，预测下一个词"——
这只需要解码器就够了。

以下是解码器Transformer的核心结构：
输入token序列
↓
[词嵌入 + 位置编码]
↓
┌─────────────────────────────┐
│ ┌─────────────────────┐ │
│ │ 掩码多头自注意力 │ │
│ └──────────┬──────────┘ │
│ ↓ (+ 残差连接) │
│ ┌──────────┴──────────┐ │
│ │ 层归一化 │ │
│ └──────────┬──────────┘ │
│ ↓ │
│ ┌──────────┴──────────┐ │
│ │ 前馈网络 │ │
│ └──────────┬──────────┘ │
│ ↓ (+ 残差连接) │
│ ┌──────────┴──────────┐ │
│ │ 层归一化 │ │
│ └──────────┬──────────┘ │
│ ↓ │
└──────── × N层（重复）────────┘
↓
[线性层 → softmax]
↓
下一个token的概率分布


接下来逐个解释每个组件。

---

## 组件一：词嵌入（Token Embedding）

输入的token序列首先被转换成向量序列。

每个token通过一个嵌入矩阵查找到对应的向量：

词表大小：50000
嵌入维度：768

嵌入矩阵 E：维度 50000 × 768
每一行是一个token的嵌入向量

token "猫" → token ID 12345 → E[12345] → [0.23, -0.87, 1.45, ..., -0.56]


这个嵌入矩阵的参数是在训练中学到的——
训练结束后，语义相近的token在嵌入空间中的向量也相近。

---

## 组件二：位置编码（Positional Encoding）

自注意力机制有一个问题：**它不知道顺序。**

在自注意力的计算中，每个位置和所有其他位置做点积，
但点积只关心向量的内容，不关心向量的位置。

"猫吃鱼"和"鱼吃猫"在自注意力看来——
如果只看内容不看位置——包含完全相同的信息。

这显然是不对的。顺序很重要。

解决方案：**在嵌入向量上加上位置信息。**

最终输入 = 词嵌入 + 位置编码

位置编码有多种实现方式：

**固定位置编码**（原始Transformer）：用三角函数生成，不需要学习。

PE(pos, 2i) = sin(pos / 10000^(2i/d))
PE(pos, 2i+1) = cos(pos / 10000^(2i/d))


不同位置的编码向量不同，模型可以通过它来区分不同位置。

**可学习的位置编码**：直接把位置编码当作可训练的参数，让模型自己学。

**旋转位置编码（RoPE）**：当前主流大模型（Llama等）使用的方式。
它不是加法而是旋转——在计算注意力点积时，
通过旋转向量来编码相对位置信息。
这使得模型能更好地捕捉相对位置关系，
并且理论上可以泛化到训练时未见过的更长序列。

---

## 组件三：掩码多头自注意力

这是Transformer最核心的计算。把它拆成两部分来理解。

### 掩码（Mask）：不能偷看未来

在语言模型中，当你在预测位置 t 的下一个词时，
你只能看到位置 1 到 t-1 的内容，不能看到位置 t+1 及之后的内容——
那是"未来"的信息，在生成时还不存在。

掩码的实现方式：在计算注意力得分之后、softmax之前，
把所有不应该看到的位置的得分设为负无穷大：
得分矩阵（softmax之前）：
位置 1 2 3 4
1 [0.5 -∞ -∞ -∞ ] ← 位置1只能看到自己
2 [0.3 0.8 -∞ -∞ ] ← 位置2能看到1和2
3 [0.1 0.4 0.7 -∞ ] ← 位置3能看到1、2、3
4 [0.2 0.6 0.3 0.9] ← 位置4能看到所有

softmax之后，-∞变成0：
1 [1.0 0 0 0 ]
2 [0.27 0.73 0 0 ]
3 [0.12 0.33 0.55 0 ]
4 [0.10 0.30 0.15 0.45]


这就是"掩码"——遮住未来的位置，确保模型只基于过去的信息做预测。

### 多头（Multi-Head）：从多个角度关注

一个自注意力头只能捕捉一种类型的关系。

但语言中的关系是多样的——
"猫坐在垫子上"这句话中：
- "猫"和"坐"有动作关系
- "猫"和"垫子"有位置关系
- "坐"和"上"有语法关系

一个注意力头很难同时捕捉所有这些关系。

多头注意力的做法：**并行运行多个注意力头，每个头独立学习关注不同类型的关系。**

输入向量 x（维度768）

拆分成12个头，每个头维度64：
头1：Q₁ = W_Q1 × x, K₁ = W_K1 × x, V₁ = W_V1 × x → 注意力输出₁
头2：Q₂ = W_Q2 × x, K₂ = W_K2 × x, V₂ = W_V2 × x → 注意力输出₂
...
头12：Q₁₂ = W_Q12 × x, K₁₂ = W_K12 × x, V₁₂ = W_V12 × x → 注意力输出₁₂

拼接所有头的输出：[输出₁; 输出₂; ...; 输出₁₂] （维度64×12=768）
再通过一个线性变换：最终输出 = W_O × 拼接结果


每个头有自己的 W_Q、W_K、W_V 矩阵，学习关注不同的模式。

在训练好的模型中研究人员发现，不同的头确实学到了不同类型的关注模式：
有的头关注语法关系，有的头关注指代关系，有的头关注近邻词。

---

## 组件四：前馈网络（FFN）

自注意力处理了位置之间的关系。
前馈网络处理**每个位置自身的特征变换**。

结构很简单——两层全连接网络：
FFN(x) = W₂ × GELU(W₁ × x + b₁) + b₂

第一层通常将维度扩大4倍（768 → 3072），
第二层再缩回来（3072 → 768）。

中间的扩大是为了给模型更多的计算空间来做特征变换。

**前馈网络在每个位置独立运行——不同位置之间不共享信息。**
位置间的信息交换全部由注意力层完成。

研究表明，前馈网络的参数中存储了大量"知识"——
可以把它理解为一种巨大的键值存储，
存储了训练数据中的模式和关联。

---

## 组件五：残差连接 + 层归一化

### 残差连接（Residual Connection）
输出 = 子层(x) + x

每个子层（注意力或FFN）的输出和输入相加。

为什么需要它？

还记得梯度消失的问题吗？
残差连接提供了一条"捷径"——梯度可以跳过子层直接流回前面。
这使得训练几十层甚至上百层的网络成为可能。

此外，残差连接让网络可以轻松学习"恒等变换"——
如果某一层不需要做任何改变，它只需要让子层输出接近零，
残差连接就会让输入原样通过。

### 层归一化（Layer Normalization）
LayerNorm(x) = γ × (x - μ) / √(σ² + ε) + β


对每个位置的向量做归一化（均值为0、方差为1），
然后乘以可学习的缩放因子 γ 和偏移 β。

作用：稳定每一层的输入分布，使训练更稳定。

---

## 组件六：输出层

最后一层Transformer块的输出经过一个线性变换，
把每个位置的向量（768维）映射到词表大小（比如50000维），
然后通过softmax得到每个token的概率分布。

logits = W_vocab × h_最后一层 （768维 → 50000维）
概率 = softmax(logits)


然后从这个概率分布中选择下一个token。

---

## 完整的数据流

把所有组件串起来，一个token序列通过Transformer的完整过程：
"今天天气很" → [token化] → [4523, 8901, 2345, 7890]
↓
[嵌入查找] → 4个768维向量
↓
[+ 位置编码] → 4个768维向量（含位置信息）
↓
[第1层]
├ 掩码多头自注意力：每个位置从前面的位置收集信息
├ + 残差连接 + 层归一化
├ 前馈网络：每个位置独立做特征变换
└ + 残差连接 + 层归一化
↓
[第2层] ... [第N层]：重复同样的结构
↓
[第N层的输出] → 取最后一个位置的向量
↓
[线性层 + softmax] → 50000维的概率分布
↓
选择概率最高的token → "好"（或从分布中采样）

## 为什么Transformer这么成功

### 1. 完全并行

没有循环结构。
训练时所有位置的计算同时进行。
这使得模型可以充分利用GPU的并行能力，
从而能在合理的时间内训练几千亿参数的模型。

### 2. 直接的长距离依赖

任意两个位置之间只有一步注意力计算的距离。
不管序列多长，位置1和位置10000之间的信息流通路径是一样短的。

### 3. 规模化能力（Scaling）

Transformer展现出了一个令人惊叹的性质——**scaling law**：
模型越大（更多参数）+ 数据越多 + 计算越多 → 性能持续提升


这个关系在很大的范围内都成立，而且目前还没有看到明确的上限。

RNN没有这个性质——因为顺序计算的瓶颈限制了它的扩展。
Transformer的并行计算架构使得它可以"吃"掉更多的参数和数据，
并且持续获得回报。

**这就是为什么当前的AI竞赛本质上是"谁能训练更大的Transformer"的竞赛。**

### 4. 架构的通用性

Transformer最初是为文本设计的，
但它的核心——自注意力——对数据类型没有假设。

只要你能把数据表示成一个"序列"（一系列向量），
Transformer就能处理它。

- 文本：token序列
- 图像：把图片切成小块（patch），每个patch作为一个token → Vision Transformer（ViT）
- 音频：把音频切成帧，每帧作为一个token
- 视频：图像帧序列
- 多模态：文本+图像的混合序列

**Transformer成为了一个统一的、跨模态的架构。**

## Transformer的局限

Transformer不是没有缺点：

### 注意力的计算复杂度

自注意力需要每个位置和所有其他位置做点积。
如果序列长度是 n，计算复杂度是 O(n²)。

当 n = 1000 时，这没问题。
当 n = 100000 时，计算量变成了 1000 万亿次运算。

**这就是"上下文窗口"限制的根本原因。**

各种技术正在解决这个问题：
- 稀疏注意力：不是所有位置都互相关注
- 线性注意力：用近似方法把复杂度降到 O(n)
- 滑动窗口注意力：每个位置只关注附近的一个窗口
- 等等

但这是一个活跃的研究领域，目前还没有完美的解决方案。

### 没有内置的"推理"能力

Transformer通过模式匹配来"推理"，不是真正的逻辑推演。
对于需要多步严格逻辑推理的问题，它仍然容易出错。

思维链（Chain of Thought）等技术通过让模型"一步步想"
来改善推理能力，但本质上仍然是引导模型走更可靠的模式匹配路径，
不是赋予它真正的逻辑推理能力。

---

## 关键认知

1. Transformer完全基于注意力，没有循环和卷积——"Attention Is All You Need"
2. 核心结构：[掩码多头自注意力 + 前馈网络 + 残差连接 + 层归一化] × N层
3. 位置编码补偿了注意力机制不感知顺序的缺陷
4. 掩码确保模型只基于过去的信息预测——不能偷看未来
5. 多头注意力让模型从多个角度关注不同类型的关系
6. Transformer的成功关键：完全并行、直接的长距离依赖、scaling law、架构通用性
7. 主要局限：O(n²)的注意力复杂度限制了上下文窗口长度
8. 当前所有主流大模型都是Transformer——理解它就理解了AI的物理结构

## 深度学习完成

你已经走完了深度学习的完整演进路径：
神经网络基础 → CNN → RNN/LSTM → 注意力机制 → Transformer


每一步都是为了解决前一步的局限：
- 全连接网络不利用数据结构 → CNN利用空间局部性
- CNN不处理序列 → RNN引入时间维度
- RNN有梯度消失和信息瓶颈 → 注意力机制直接访问所有历史位置
- 注意力在RNN中仍受顺序限制 → Transformer完全抛弃循环
