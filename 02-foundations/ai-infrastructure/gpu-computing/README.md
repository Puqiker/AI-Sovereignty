# GPU计算

> 你已经知道神经网络的核心运算是矩阵乘法。
> 这个模块解释为什么矩阵乘法让GPU成为AI的标准硬件，
> 以及GPU计算的基本特性如何影响你对AI系统的理解。

---

## CPU vs GPU：设计哲学的根本差异

### CPU：少量核心，每个核心很强

CPU有几个到几十个核心（比如8核、16核）。
每个核心非常复杂，能独立执行复杂的逻辑判断、分支预测、乱序执行。
擅长**串行处理复杂任务**——一个任务接一个任务地高效完成。

类比：CPU是一个由8名博士组成的团队。
每个人能独立解决复杂问题，但同一时刻只能做8件事。

### GPU：大量核心，每个核心较简单

GPU有几千到几万个核心（NVIDIA A100有6912个CUDA核心）。
每个核心比CPU核心简单得多，不擅长复杂逻辑。
擅长**并行处理大量简单任务**——几千个核心同时做相同类型的运算。

类比：GPU是一个由5000名小学生组成的团队。
每个人只能做简单加减乘除，但5000人同时算，总吞吐量极高。

### 为什么AI选择GPU

神经网络的核心运算——矩阵乘法——是这样的：
C[i][j] = Σ_k A[i][k] × B[k][j]

矩阵C的每个元素是A的一行和B的一列的点积。
关键性质：**C的每个元素的计算是独立的。**

一个 1000×1000 的矩阵乘法有 100万个元素需要计算，
这100万个计算互相不依赖——完美适合GPU的并行架构。

CPU算这个矩阵乘法：8个核心轮流算100万个元素。
GPU算这个矩阵乘法：几千个核心同时算，速度快几百倍。

**AI的计算瓶颈是大规模矩阵运算，GPU天生就是为这种运算设计的。**

## GPU的关键指标

当你看到GPU的规格时，需要关注的关键数字：

### 算力（FLOPS）

每秒能做多少次浮点运算。
NVIDIA A100：312 TFLOPS（FP16）= 每秒312万亿次运算
NVIDIA H100：989 TFLOPS（FP16）= 每秒989万亿次运算

### 显存（GPU Memory）

GPU上的内存大小。模型参数、KV Cache、中间计算结果都存在这里。

A100：40GB 或 80GB
H100：80GB
消费级 RTX 4090：24GB


**显存大小直接决定了你能运行多大的模型。**

一个粗略的估算：
- 模型每个参数占2字节（FP16精度）
- 一个7B参数的模型至少需要 7B × 2 = 14GB 显存
- 加上KV Cache和中间计算，实际需要约20-24GB
- 一个70B参数的模型需要约140GB——一张消费级GPU装不下

### 内存带宽（Memory Bandwidth）

显存和计算核心之间的数据传输速度。

A100：2 TB/s
H100：3.35 TB/s
RTX 4090：1 TB/s


你在推理机制模块学过：decode阶段是内存带宽密集型的。
带宽越高，每个token的生成速度越快。

## 精度与量化

### 浮点精度

FP32（32位浮点）：精度最高，每个参数4字节，但慢
FP16（16位浮点）：精度够用，每个参数2字节，速度快一倍
BF16（Brain Float 16）：Google设计的16位格式，范围更大但精度略低
INT8（8位整数）：精度更低，每个参数1字节，推理速度快
INT4（4位整数）：精度最低，每个参数0.5字节，速度最快但质量有损失


### 量化

**量化就是用更低精度的数字来表示模型参数，换取更小的内存占用和更快的推理速度。**

一个70B参数的模型：
- FP16：140GB → 至少需要2张A100（80GB）
- INT8：70GB → 1张A100就够了
- INT4：35GB → 可能塞进一张消费级RTX 4090

量化会损失一些模型质量，但研究表明：
- INT8量化通常几乎不影响模型质量
- INT4量化会有轻微的质量下降，但在大多数场景中可接受

**量化是让大模型在有限硬件上运行的关键技术。**
这在你后面学习私有大模型部署专题时会再次用到。

---

## 关键认知

1. GPU适合AI是因为矩阵运算天然可并行——几千个核心同时计算
2. 显存大小决定能运行多大的模型——这是最直接的硬件限制
3. 内存带宽影响推理速度——特别是decode阶段
4. 量化用精度换空间——让大模型在小显存上运行的关键技术
5. 理解这些让你能判断：某个模型能不能在你的硬件上运行，以及性能瓶颈在哪
