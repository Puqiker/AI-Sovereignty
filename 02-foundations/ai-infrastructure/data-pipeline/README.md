# 数据管线

> 模型的能力上限由数据决定。
> 数据管线就是把原始的、杂乱的数据变成可以用于训练的、干净的数据集的完整流程。
> 这个过程的工程复杂度常常被低估——
> 很多AI项目失败不是因为模型不好，而是因为数据没处理好。

---

## 数据管线的完整流程
原始数据（网页、文档、数据库……）
↓
[数据收集] ← 爬虫、API、数据库导出、人工收集
↓
[数据清洗] ← 去除噪声、修复格式、处理缺失值
↓
[去重] ← 移除重复或高度相似的数据
↓
[质量过滤] ← 移除低质量内容（乱码、广告、极短文本等）
↓
[有害内容过滤] ← 移除违法、色情、极端有害的内容
↓
[隐私处理] ← 脱敏、移除可识别个人信息
↓
[分词/格式化] ← 转换为模型训练需要的格式
↓
[数据分割] ← 划分训练集/验证集/测试集
↓
可用于训练的数据集

## 每个环节的关键挑战

### 数据清洗

原始网页数据中充斥着：
- HTML标签、CSS样式、JavaScript代码
- 导航栏、广告、侧边栏内容
- 编码错误导致的乱码
- 格式不一致（日期、数字、单位的不同表示方式）

清洗的目标是提取出纯净的文本内容。
这个过程看似简单，但在互联网规模上需要大量的工程努力。

### 去重

互联网上大量内容是重复的——转载、抄袭、模板生成。
如果不去重，模型会在重复内容上过度训练，导致输出中也倾向于重复这些内容。

去重的方法：
- **精确去重**：完全相同的文本（哈希比对）
- **近似去重**：高度相似的文本（MinHash、SimHash等算法）
- **段落级去重**：即使文档整体不同，移除出现在大量文档中的相同段落

### 质量过滤

不是所有的文本都有训练价值。
低质量文本（乱码、纯列表、自动生成的垃圾内容）会拉低模型质量。

常见的质量过滤指标：
- 文本长度（太短的可能没有实质内容）
- 语言检测（确保是目标语言）
- 困惑度过滤（用一个已有的语言模型评估文本的"自然度"）
- 关键词过滤（特定的垃圾内容标记词）

### 数据配比

训练数据通常来自多个来源——网页、书籍、代码、论文等。
不同来源的数据比例会影响模型的能力分布。
代码数据多 → 模型编程能力更强
学术论文多 → 模型学术写作能力更强
对话数据多 → 模型对话能力更强


这个配比是一个关键的设计决策——
没有"最优"的配比，取决于你想要模型擅长什么。

## 数据管线的持续性

数据管线不是一次性工作。

- 训练数据需要定期更新（世界在变化）
- 新的数据源需要被纳入
- 数据质量问题需要持续监控
- 用户反馈可能揭示训练数据中的问题

**数据管线是一个持续运行的系统，不是一个一次性的脚本。**

---

## 关键认知

1. 数据管线 = 从原始数据到可训练数据集的完整工程流程
2. 每个环节（清洗、去重、过滤、脱敏）都有独特的挑战和方法
3. 数据配比是影响模型能力分布的关键设计决策
4. 数据管线是持续性工作——不是一次做完就不管了
5. 很多AI项目失败的根因在数据，不在模型
