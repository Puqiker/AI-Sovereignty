# 分布式训练

> 一张GPU装不下一个大模型的参数，更不用说训练时需要的梯度和优化器状态。
> 训练一个大语言模型需要把工作分散到几百甚至几千张GPU上。
> 这就是分布式训练。

---

## 为什么需要分布式

训练时的内存需求远超推理：
训练一个7B参数的模型（FP16）需要存储：

- 模型参数：14GB
- 梯度：14GB（每个参数对应一个梯度值）
- 优化器状态（Adam）：56GB（每个参数需要存储一阶和二阶动量）
- 激活值（中间计算结果，用于反向传播）：取决于batch size，可能几十GB
总计：可能需要100GB+——一张GPU装不下

对于175B参数的模型，这些数字全部乘以25，总需求达到数TB——
必须用分布式方案。

---

## 三种基本并行策略

### 数据并行（Data Parallelism）
每张GPU都有模型的完整副本。
每张GPU处理不同的训练数据批次。
计算完梯度后，所有GPU之间同步梯度（取平均）。
每张GPU用同步后的梯度更新各自的模型参数。


优点：实现简单，通信开销相对可控。
局限：每张GPU都需要能装下完整的模型——对于大模型不可行。

### 模型并行（Model Parallelism / Tensor Parallelism）

模型的一层被切分到多张GPU上。
一个矩阵乘法被拆成多个小的矩阵乘法，分别在不同GPU上计算。
每一步计算都需要GPU之间通信来交换中间结果。


优点：可以训练单张GPU装不下的模型。
局限：通信开销大——每一层都需要GPU之间同步。

### 流水线并行（Pipeline Parallelism）

模型的不同层放在不同GPU上。
GPU 1负责第1-10层，GPU 2负责第11-20层，……

数据像流水线一样流过各个GPU：
GPU 1处理batch 1 → 把结果传给GPU 2 → GPU 1开始处理batch 2
GPU 2处理batch 1 → 传给GPU 3 → GPU 2处理batch 2


优点：单张GPU只需要存储模型的一部分，内存需求降低。
局限：存在"流水线气泡"——开头和结尾阶段有GPU在空闲等待。

### 实际中的混合策略

大规模训练通常同时使用三种并行策略：
整体架构：

- 几千张GPU分成多个组
- 组内使用模型并行（同一层的矩阵运算分散到组内各GPU）
- 组之间使用流水线并行（不同的层放在不同组）
- 在所有组上使用数据并行（不同组处理不同的数据批次）


协调这些策略需要精密的通信调度和极高的工程能力。

---

## 通信是瓶颈

分布式训练的最大挑战不是计算，而是**通信**。

GPU之间交换数据的速度远低于GPU本身的计算速度。
如果通信设计不好，GPU大部分时间在等待数据而不是在计算。

关键通信技术：
- **NVLink**：NVIDIA的GPU间高速互连，带宽远高于PCIe
- **InfiniBand**：跨机器的高速网络，大规模集群的标准选择
- **梯度压缩**：压缩需要传输的梯度数据，减少通信量

---

## 关键认知

1. 训练时的内存需求远超推理——参数+梯度+优化器状态+激活值
2. 三种并行策略各有优劣——实际中混合使用
3. 通信是分布式训练的核心瓶颈——不是GPU不够快，而是数据传输不够快
4. 大规模分布式训练需要极高的工程能力——不仅仅是算法问题
5. 理解这些让你能估算训练成本、判断私有部署的硬件需求
