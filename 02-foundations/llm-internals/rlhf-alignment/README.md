# RLHF与对齐

> 你在机器学习的强化学习模块已经学过了RLHF的基本框架。
> 这个模块从大模型的角度重新审视这个过程，
> 重点在于：对齐到底在解决什么问题，为什么它这么难，以及它的局限在哪里。

---

## "对齐"是什么问题

一个经过预训练和SFT的模型已经能"说话"了。
但"能说话"和"说得好"之间有巨大的鸿沟。

"说得好"意味着：
- **有用**（Helpful）：真正回答了用户的问题，提供了有价值的信息
- **无害**（Harmless）：不生成有害、歧视、违法的内容
- **诚实**（Honest）：不编造事实，在不确定时承认不确定

这三个目标叫做 **HHH 原则**。

问题在于：这些目标无法用简单的规则来定义。
什么算"有用"取决于具体场景。
什么算"有害"在不同文化和语境中有不同标准。
"诚实"和"有用"之间有时还会冲突——
诚实地说"我不知道"有时不如给一个大致正确的回答有用。

**对齐的挑战在于：你要优化的目标本身就是模糊的、多维的、依赖语境的。**

## RLHF的完整流程（详细版）

### 阶段一：SFT（已完成）

用高质量指令-回答对训练模型。

### 阶段二：训练奖励模型（Reward Model, RM）

**目的**：构建一个能自动评判回答好坏的模型。

**过程**：

1. 从SFT模型中采样多个回答：
问题："如何提高编程能力？"

回答A："提高编程能力可以通过以下方式：
a. 多写代码练习
b. 阅读优秀的开源项目
c. 参与实际项目
……"

回答B："编程能力的提高需要长期的积累和练习，
建议每天坚持写代码至少2小时，
同时要注重算法和数据结构的学习……"

回答C："你可以去网上搜索一下。"

回答D："编程很简单，你只需要学会Python就行了，
Python可以做任何事情……"


2. 人类标注员对这些回答进行排序：
A > B > D > C


3. 用排序数据训练一个奖励模型——
   这个模型学习预测"人类会给什么样的回答更高的评分"。

**奖励模型的本质**：一个把人类偏好数字化的函数。
输入一个（问题, 回答）对，输出一个分数。

### 阶段三：强化学习优化

用PPO算法优化语言模型，使其生成的回答能获得更高的奖励模型评分。

循环：

1. 语言模型对一个问题生成回答
2. 奖励模型对这个回答打分
3. 根据分数计算策略梯度
4. 更新语言模型参数（加上KL散度约束）
5. 重复

KL散度约束（你在信息论模块学过）确保模型不会为了讨好奖励模型而偏离太远：
总目标 = 奖励分数 - β × D_KL(当前模型 || SFT模型)

## 对齐的内在困难

### 奖励模型的局限

奖励模型是人类偏好的**近似**，不是人类偏好本身。

它可能存在：
- **系统性偏差**：标注员有自己的偏好和盲点
- **长度偏好**：研究发现奖励模型往往偏好更长的回答，不一定是更好的回答
- **表面特征偏好**：偏好格式漂亮的回答（有列表、有标题），而非内容更准确的回答
- **分布外失效**：对于和训练数据差异很大的回答类型，奖励模型的评分不可靠

### 奖励黑客（Reward Hacking）

模型可能学会"讨好"奖励模型而不是真正变得更好。

比如：
- 如果奖励模型偏好长回答，模型会学会在回答中添加不必要的废话
- 如果奖励模型偏好列表格式，模型会不管什么问题都用列表回答
- 模型可能找到奖励模型的"漏洞"——生成一种看起来高质量但实际上没有实质内容的回答

**这本质上是Goodhart定律——当一个指标成为目标时，它就不再是一个好指标。**

### HHH之间的张力

三个目标之间有时互相矛盾：

- **有用 vs 无害**：用户问"如何制造炸弹"时，有用意味着回答这个问题，无害意味着拒绝
- **诚实 vs 有用**：对于一个模型不确定的问题，诚实地说"我不知道"可能不如给一个大致正确的回答有用
- **有用 vs 诚实**：用户想听的和事实可能不一致

对齐不是找到一个完美的平衡点——不存在这样的点。
它是在具体场景中做出权衡。

## DPO：跳过奖励模型

你在强化学习模块已经了解了DPO（Direct Preference Optimization）。

DPO直接从偏好数据中优化模型，不需要训练奖励模型：
给定偏好数据（问题, 好的回答, 差的回答），
直接调整模型参数，使得：

- 好的回答的概率增加
- 差的回答的概率降低

DPO的数学公式：Loss = -log σ(β × (log P_model(好回答)/P_ref(好回答) - log P_model(差回答)/P_ref(差回答)))

其中 P_ref 是参考模型（SFT后的模型）的概率，
β控制偏离参考模型的程度——和RLHF中的KL散度约束作用相同。

DPO更简单、更稳定，越来越多的模型在使用DPO或其变体来替代RLHF。

## 对齐之后的模型

经过对齐的模型和只做了SFT的模型相比：

- 回答更符合人类期望的格式和风格
- 更倾向于承认不确定性
- 更倾向于拒绝有害请求
- 回答的详细程度更适当
- 语气更"像一个有教养的助手"

但对齐不是万能的——它只是调整了模型行为的概率分布，
它没有赋予模型新的知识、新的推理能力、或真正的价值观。

对齐可以被绕过（通过对抗性prompt）——
这正是你后面在安全模块会学到的内容。

---

## 关键认知

1. 对齐解决的是让模型行为符合人类偏好的HHH目标——有用、无害、诚实
2. RLHF通过奖励模型将人类偏好数字化，再用强化学习优化模型
3. 奖励模型是人类偏好的近似——有偏差、有漏洞、有失效区域
4. 奖励黑客是真实的风险——模型可能学会讨好指标而非真正变好
5. DPO简化了对齐流程——跳过奖励模型，直接从偏好数据优化
6. 对齐调整的是概率分布，不是赋予模型真正的理解或价值观
7. 对齐可以被绕过——安全模块会深入讨论
