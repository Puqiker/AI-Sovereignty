# 上下文窗口与记忆

> 你可能已经注意到：和AI对话时，如果对话太长，
> 模型似乎会"忘记"前面说过的内容。
> 这不是模型在偷懒——这是一个物理限制。
> 理解这个限制的本质，让你能更好地管理和AI的对话。

---

## 什么是上下文窗口

上下文窗口（Context Window）是模型一次能处理的**最大token数量**。
GPT-3.5：4096 tokens（后来扩展到16K）
GPT-4：8192 / 32K / 128K tokens
Claude：100K / 200K tokens
Llama 3：8K / 128K tokens


这个数字包括**输入和输出的总和**。
如果上下文窗口是8K，你的输入用了6K token，
模型最多只能生成2K token的回答。

---

## 为什么存在这个限制

### 计算复杂度

你在Transformer模块学过：自注意力的计算复杂度是O(n²)。
n = 4K（4096 tokens）：约1600万次注意力计算
n = 128K（131072 tokens）：约170亿次注意力计算

序列长度增加32倍，计算量增加了约1000倍。

### 内存需求

KV Cache的大小随序列长度线性增长：
一个典型的70B参数模型：
每个token的KV Cache ≈ 每层2个矩阵 × 80层 × 8192维 × 2字节 ≈ 2.6MB

128K token的KV Cache ≈ 128K × 2.6MB ≈ 333GB

这个内存需求是巨大的——需要多张高端GPU来存放。

### 训练时的限制

模型在训练时使用的最大序列长度决定了它能有效处理的上下文长度。
如果模型在训练时只见过4K长度的序列，
让它处理128K长度的序列可能表现很差——
因为位置编码、注意力模式等都没有在这个长度上被训练过。

扩展上下文窗口需要额外的训练或特殊技术（如RoPE的频率外推）。

---

## 上下文窗口内的"记忆"

在上下文窗口范围内，模型的"记忆"是怎么工作的？

**注意力就是记忆。**

当模型在生成第100个token时，
它通过注意力机制可以"回看"前面99个token中的任何一个。
这就是模型在上下文窗口内的"记忆"机制。

但这种记忆有特点：

### 不是完美的记忆

虽然注意力理论上可以关注任何位置，
但实际训练出来的模型不会均匀地关注所有位置。

研究发现存在**"中间遗失"（Lost in the Middle）**现象：
模型对上下文开头和结尾的信息记得最好，
对中间部分的信息注意力较弱。

信息位置 → 模型的注意力
开头：████████████ 高
中间：████ 低
结尾：████████████ 高


这意味着：如果你把最重要的信息放在输入的中间部分，
模型可能不会给它足够的注意力。

### 不是累积的记忆

每一次推理调用，上下文窗口都是独立的。
模型不会把上一次对话的内容自动带到下一次。

你和AI的"多轮对话"，实际上是应用层在做的事情——
每次新的对话轮次，应用把之前所有的对话历史拼接起来作为新的输入。

第1轮：用户问题1 → 模型回答1
第2轮：用户问题1 + 模型回答1 + 用户问题2 → 模型回答2
第3轮：用户问题1 + 模型回答1 + 用户问题2 + 模型回答2 + 用户问题3 → 模型回答3
...


这就是为什么长对话会越来越慢——每一轮的输入越来越长。
也是为什么长对话最终会"忘记"——当总token数超过上下文窗口时，
最早的对话内容必须被截断丢弃。

---

## 更长的上下文窗口 ≠ 更好

一个常见的误解是：上下文窗口越大越好。

实际上：

### 1. 检索精度下降

上下文越长，模型需要从中"找到"相关信息的难度越大。
就像你在一本10页的文档中找一个信息很容易，
但在一本1000页的文档中找同一个信息就困难得多。

### 2. 成本增加

更长的上下文意味着更多的计算和内存——更慢、更贵。

### 3. 不相关信息的干扰

把大量不相关的内容塞进上下文，
可能反而降低模型对重要信息的注意力——
注意力是一个加权分配机制，
当你给它太多东西"注意"时，分配给每个东西的注意力就减少了。

**更好的策略通常不是"往上下文里塞更多东西"，
而是"精选最相关的信息放进上下文"。**

这就是RAG（检索增强生成）的核心思想——
先用检索系统找到最相关的文档片段，
只把这些片段放进上下文，而不是把整个知识库塞进去。

---

## 模型真的有"记忆"吗

在严格意义上：**当前的大语言模型没有持久记忆。**

- **上下文内记忆**：通过注意力机制在上下文窗口内"记住"信息。临时的，对话结束就消失。
- **参数记忆**：训练过程中压缩进参数里的统计模式。持久的，但不可控、不可更新。
- **外部记忆**：通过RAG等机制从外部数据源检索信息。可更新，但依赖检索质量。

**模型不会因为和你对话而"学到"新东西。**
你告诉模型的信息存在于当前对话的上下文中，
模型关闭后这些信息就消失了——除非被外部系统保存。

---

## 关键认知

1. 上下文窗口是模型一次能处理的最大token数——输入+输出的总和
2. 限制来源于注意力的O(n²)复杂度和KV Cache的内存需求
3. "中间遗失"效应——模型对开头和结尾的信息记得更好
4. 多轮对话是应用层拼接实现的——模型自己没有跨轮次的记忆
5. 更长的上下文不一定更好——精选相关信息比塞更多内容有效
6. 模型没有持久记忆——不会因为对话而"学到"新东西
