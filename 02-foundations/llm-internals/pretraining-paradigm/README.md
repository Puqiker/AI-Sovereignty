# 预训练范式

> 一个刚初始化的Transformer，参数全是随机数字，输出的也是随机的胡话。
> 预训练就是让这个"什么都不会"的模型，
> 通过阅读互联网上几乎所有的文本，变成一个"什么都知道一点"的基础模型。
> 这个过程的规模、成本和工程复杂度都是惊人的。

---

## 预训练的任务：下一个词预测

你在机器学习模块已经知道了：
语言模型的预训练本质上是一个超大规模分类任务——给定前文，预测下一个token。
输入："人工智能的核心技术"
目标：预测下一个token是"包括"（或"是"或"有"……的概率分布）

具体来说，对于训练数据中的每一段文本，
模型在每个位置都要预测下一个token：
"人工智能的核心技术包括机器学习"

位置1：给定"人工" → 预测"智能"
位置2：给定"人工智能" → 预测"的"
位置3：给定"人工智能的" → 预测"核心"
...

每一个位置就是一个训练样本。
一段1000个token的文本提供了999个训练样本。

当你在几万亿token的文本上训练时，
模型获得了几万亿个训练样本。

### 损失函数

你在数学基础信息论模块学过——交叉熵损失：
Loss = -1/N × Σᵢ log P(真实tokenᵢ | 前文ᵢ; θ)

对于每个位置，模型给真实的下一个token分配的概率越高，损失越小。
训练的目标就是最小化这个损失——让模型在所有训练数据上的预测尽可能准确。

## 训练数据：模型见过什么

预训练数据的规模和质量直接决定了模型的能力边界。

### 数据来源

典型的预训练数据包括：

- **网页文本**：Common Crawl等项目抓取的互联网网页，是最大的数据来源
- **书籍**：各类书籍的数字化文本
- **学术论文**：arXiv等平台的论文
- **代码**：GitHub上的开源代码
- **维基百科**：高质量的百科全书文本
- **社交媒体/论坛**：Reddit等平台的讨论内容

### 数据规模

GPT-3（2020）：约3000亿token
Llama 2（2023）：约2万亿token
当前最大规模的模型：据估计在10万亿token以上


做一个对比：一个人如果每天阅读8小时、每分钟阅读300字，
一年能读大约5000万字（约5000万token）。
读60年是30亿token——大约是GPT-3训练数据的1%。

**模型在预训练中"阅读"的文本量，远超任何个人一生能阅读的总量。**

### 数据质量

"数据多"不等于"数据好"。

互联网上充斥着低质量内容、重复内容、错误信息、有害内容。
如果模型在这些数据上训练，它也会学到这些模式。

数据预处理通常包括：
- **去重**：移除重复或高度相似的文本
- **质量过滤**：基于规则或分类器过滤低质量内容
- **有害内容过滤**：移除极端有害的内容
- **语言识别和分类**：按语言、主题分类数据
- **个人信息脱敏**：尝试移除可识别的个人信息（但无法保证完全移除）

**数据预处理的质量往往比模型架构的选择更重要。**

## 训练过程：规模化的挑战

### 计算规模

训练一个大语言模型需要的算力是惊人的：
GPT-3（175B参数）：约3.14 × 10²³ 次浮点运算
需要约1000块A100 GPU训练约1个月
训练成本估计在数百万到上千万美元

更大的模型：成本更高，可能需要数万块GPU运行数月

### 分布式训练

没有任何单张GPU能装下一个几千亿参数的模型。

分布式训练策略：

- **数据并行**：每张GPU有模型的完整副本，但各自处理不同的数据批次。梯度在GPU之间同步。
- **模型并行**：模型太大放不进一张GPU，把模型切分到多张GPU上。一次前向传播需要GPU之间通信。
- **流水线并行**：把模型的不同层放在不同GPU上，数据像流水线一样流过各个GPU。

实际的大规模训练通常同时使用多种并行策略。

### 训练的不稳定性

大规模训练容易出现各种问题：

- **损失突然飙升（loss spike）**：训练过程中损失突然大幅上升，可能由学习率过高、数据中的异常样本、或数值计算不稳定导致
- **训练崩溃**：参数值变成NaN（不是数字），训练无法继续
- **硬件故障**：几千张GPU连续运行几个月，硬件故障是常态而非例外

解决这些问题需要大量的工程经验——
检查点保存（定期保存模型状态以便从故障中恢复）、
学习率调度策略、梯度裁剪、混合精度训练等等。

## 预训练学到了什么

当预训练完成后，模型学到了什么？

### 语言模式

最直接的：词与词之间的统计关联。
"中华人民___" → 极高概率预测"共和国"。

### 语法结构

通过大量文本中一致的语法模式，
模型隐式地学会了语法规则——
不是作为显式的规则，而是作为参数中的统计模式。

### 世界知识

训练数据中包含了大量的事实性描述：
"地球绕太阳公转""水的分子式是H₂O"……

这些事实以统计模式的形式被压缩进了模型参数中。
但注意——这是压缩后的统计模式，不是精确的知识库。
这就是为什么模型有时会把事实搞混或编造事实。

### 推理模式

训练数据中包含大量的逻辑推理过程——
数学推导、论证过程、问题分析。
模型从中学到了"推理的模式"——
什么样的推理步骤通常跟在什么样的前提之后。

但这仍然是模式匹配，不是真正的逻辑推理。
模型可以复现训练数据中常见的推理模式，
但面对需要全新推理路径的问题，它可能失败。

### 格式和风格

模型学会了各种文本格式和写作风格——
新闻报道的写法、学术论文的格式、代码的语法、诗歌的韵律。

## 预训练模型的局限

经过预训练的模型（也叫基础模型或基座模型）有几个重要的局限：

### 1. 它只会"续写"

预训练目标是"预测下一个词"。
模型学会的是：给定一段文字，接着往下写。

如果你给它"问：中国的首都是哪里？"，
它可能接着写"答：北京。问：美国的首都是哪里？答：……"——
因为在训练数据中，这种问答格式后面通常跟着更多问答。

它不会自动理解"你是在和它对话，它应该只回答这一个问题"。
让它变成一个有用的助手，需要后续的微调和对齐。

### 2. 没有安全边界

预训练模型学到了训练数据中的所有模式——包括有害的、不当的模式。
它可能生成歧视性内容、有害指令、虚假信息。

安全边界在预训练阶段几乎不存在，需要通过RLHF等对齐过程来建立。

### 3. 知识有截止日期

模型的知识来自训练数据，训练数据有收集截止日期。
模型不知道截止日期之后发生的事情。

## Scaling Law：越大越强

预训练阶段最重要的经验发现之一是**scaling law**：
模型性能和三个因素的关系近似于幂律函数：

模型大小（参数量）
数据量
计算量
三者中的任何一个增加，性能都会提升（损失下降）
而且在很大的范围内，这种关系是可预测的


这意味着：
- 在一定范围内，你可以通过增大模型和数据来可预测地提升性能
- 你可以用小规模的实验来预测大规模训练的结果
- 目前还没有看到明确的"天花板"——性能还在随规模增长而提升

**这就是为什么AI竞赛在很大程度上变成了资源竞赛——谁有更多的GPU和数据，谁就能训练更好的模型。**

但scaling law也有它的边界——越到后面，每提升一点性能需要的额外资源越多。
而且有些能力（如可靠的数学推理）似乎不能仅靠规模来获得。

---

## 关键认知

1. 预训练 = 在海量文本上做"预测下一个token"的任务
2. 训练数据的质量和规模直接决定模型能力的上限
3. 大规模训练需要分布式计算、工程稳定性保障、巨额成本
4. 模型学到的是统计模式（语言、知识、推理模式），不是真正的理解
5. 预训练后的模型只会"续写"，需要微调和对齐才能变成有用的助手
6. Scaling law表明性能随规模可预测地提升——但成本也在指数增长
