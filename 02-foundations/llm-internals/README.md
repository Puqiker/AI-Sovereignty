# 大模型内核

> 你已经理解了Transformer的架构——大语言模型的物理结构。
> 这个模块回答的是：**这个结构是怎么变成一个"能说话"的系统的？**
>
> 从文字如何变成数字，到模型如何被训练，
> 到模型如何被调整以符合人类偏好，
> 到模型推理时实际发生了什么，
> 到它为什么会幻觉，
> 到你如何基于原理和它有效沟通。
>
> 这是一条完整的生命线——大语言模型从诞生到运行的全过程。

---

## 学习顺序

按照大模型生命周期的顺序来学习：

| 顺序 | 模块 | 核心问题 |
|------|------|---------|
| 1 | [分词与嵌入](tokenization-embedding/) | 文字是怎么变成模型能处理的数字的 |
| 2 | [预训练范式](pretraining-paradigm/) | 模型是怎么在海量数据上学会"语言"的 |
| 3 | [微调方法](fine-tuning-methods/) | 怎么把一个通用模型变成特定任务的专家 |
| 4 | [RLHF与对齐](rlhf-alignment/) | 怎么让模型从"能说话"变成"说人话" |
| 5 | [推理机制](inference-mechanism/) | 模型回答你的问题时，内部到底发生了什么 |
| 6 | [上下文窗口与记忆](context-window-memory/) | 模型的"记忆"有多大、怎么工作、有什么限制 |
| 7 | [幻觉分析](hallucination-analysis/) | 模型为什么会一本正经地编造内容 |
| 8 | [基于原理的Prompt Engineering](prompt-engineering-from-principles/) | 理解原理之后，怎么和模型有效沟通 |

大模型的生命线：

文字 → [分词与嵌入] → 数字
↓
[预训练] → 学会语言模式的基础模型
↓
[微调] → 适配特定任务的模型
↓
[RLHF对齐] → 符合人类偏好的模型
↓
[推理] → 用户输入 → 模型输出
↓
理解 [上下文窗口] [幻觉] [Prompt设计]
→ 你能驾驭这个模型

---

## 完成标准

当你完成大模型内核模块后，你应该能够：

- [ ] 解释一段文字从输入到模型输出的完整数据流——每一步发生了什么
- [ ] 理解预训练、微调、RLHF三个阶段各自解决什么问题
- [ ] 解释模型为什么会幻觉——从概率机制的角度，不是用"它不聪明"来解释
- [ ] 知道上下文窗口的物理限制来自哪里，以及为什么"塞更多内容"不总是更好
- [ ] 基于对模型工作原理的理解来设计Prompt，而不是背诵Prompt技巧

完成后，继续进入 [AI基础设施](../ai-infrastructure/)。
