# 强化学习

> 监督学习：有人给你答案，你照着学。
> 无监督学习：没人给答案，你自己发现结构。
> 强化学习：没有标准答案，但有人告诉你"这次做得好"或"这次做得不好"。
> 你在这个奖惩信号中摸索出最优策略。
>
> 理解强化学习是理解RLHF（人类反馈强化学习）的前提，
> 而RLHF是当前大语言模型从"能说话"变成"说人话"的关键一步。

---

## 为什么需要强化学习

先想一个问题：

当你训练一个语言模型做"预测下一个词"的任务时（监督学习），
模型会学到语言的统计模式——什么词经常跟在什么词后面。

但这并不意味着模型的输出是**有用的、安全的、符合人类偏好的**。

一个只做了预训练的模型可能会：
- 生成语法完美但内容有害的文本
- 回答问题时编造事实（因为编造的文本在统计上也是"合理"的）
- 在你问它简单问题时给出冗长啰嗦的回答
- 在需要拒绝的场景中继续回答

这些问题没法用监督学习直接解决，因为：
- 你没法为每一种可能的输入标注一个"完美回答"
- "好的回答"不是一个固定的标签，它取决于上下文、用户需求、安全标准等复杂因素
- 你需要的不是让模型预测"正确的下一个词"，而是让模型生成"人类整体偏好的回答"

**强化学习解决的就是这类问题——不是告诉模型"答案是什么"，而是告诉它"这个做得好，那个做得不好"，让它自己学会什么策略能获得最多的"好评"。**

## 强化学习的基本框架

强化学习有四个核心概念：
智能体（Agent） ←→ 环境（Environment）
↓ ↓
动作（Action） 状态（State）+ 奖励（Reward）


1. **智能体（Agent）**：做决策的主体。在RLHF中，智能体就是语言模型。

2. **环境（Environment）**：智能体所处的世界。在RLHF中，环境包括用户的输入和奖励模型的反馈。

3. **状态（State）**：环境的当前情况。在语言模型中，状态就是当前的对话上下文。

4. **动作（Action）**：智能体在某个状态下可以采取的行为。在语言模型中，动作就是生成下一个token。

5. **奖励（Reward）**：环境对智能体动作的反馈信号。在RLHF中，奖励来自一个**奖励模型**——这个模型被训练来模拟人类对回答质量的判断。

6. **策略（Policy）**：智能体的行为模式——在什么状态下采取什么动作。语言模型本身就是一个策略：给定当前上下文（状态），输出下一个token的概率分布（动作的概率）。

### 学习过程

1. 智能体观察当前状态
2. 根据当前策略选择一个动作
3. 环境返回新的状态和一个奖励信号
4. 智能体根据奖励调整策略——增加获得高奖励的动作的概率，减少获得低奖励的动作的概率
5. 重复


**目标：找到一个策略，使得长期累积奖励最大化。**

## RLHF：强化学习在大语言模型中的应用

现在你有了强化学习的框架，来看它如何被用来"对齐"大语言模型。

RLHF的完整流程分三步：

### 第一步：监督微调（SFT）

先用人类标注的高质量对话数据对预训练模型做监督学习微调。

输入："解释一下量子计算"
人类写的高质量回答："量子计算是一种利用量子力学原理进行计算的技术..."


这一步让模型从"预测下一个词"变成"像人类助手一样回答问题"。

但光靠SFT不够——人类不可能为每一种问题都写一个标准答案。

### 第二步：训练奖励模型（RM）

让模型对同一个问题生成多个不同的回答。
然后让人类标注员对这些回答进行排序——哪个更好、哪个更差。
问题："解释一下量子计算"

回答A：[一段清晰、准确、适度详细的解释]
回答B：[一段啰嗦、有错误的解释]
回答C：[一段太简短、不够有用的解释]

人类排序：A > C > B


用这些排序数据训练一个**奖励模型**——
这个模型学会了预测"人类会给什么样的回答更高的分数"。

奖励模型的本质是一个**人类偏好的数学近似**。

### 第三步：用强化学习优化模型

现在把所有东西连起来：
智能体 = 语言模型
状态 = 用户问题 + 已生成的部分回答
动作 = 生成下一个token
奖励 = 奖励模型对完整回答的评分


使用强化学习算法（通常是PPO——Proximal Policy Optimization）来调整语言模型的参数，
使得它生成的回答能获得更高的奖励模型评分。

**但有一个关键约束**——还记得数学基础信息论模块讲的KL散度吗：
目标 = 最大化奖励 - β × D_KL(当前模型 || 原始模型)


这个KL散度惩罚项防止模型为了讨好奖励模型而偏离太远——
如果没有这个约束，模型可能会学到一些"讨巧"的输出模式
（比如输出特别长的回答来刷奖励分），
但实际上丧失了基本的语言能力。

### RLHF的效果

经过RLHF训练后的模型，和只做了预训练的模型相比：
- 回答更符合人类的期望格式和风格
- 更倾向于承认自己不知道，而不是编造答案
- 更倾向于拒绝有害请求
- 回答的详细程度更适中

**RLHF是让大语言模型从"能说话"变成"说人话"的关键技术。**

## 强化学习的独特性

强化学习和监督学习有几个根本性的区别：

### 1. 延迟奖励

在监督学习中，每一步都有即时的反馈——预测对了还是错了。
在强化学习中，奖励可能是延迟的。

在语言模型中，奖励通常在整个回答生成完毕后才给出。
模型在生成每一个token时并不知道这个token对最终奖励的贡献是什么。
这就需要算法能把最终奖励"分配"回每一步的决策。

### 2. 探索与利用的权衡

模型应该利用当前已知的好策略（生成已知能得高分的回答类型），
还是探索新的策略（尝试不同的回答方式，看看是否能发现更好的）？

太多利用，模型会陷入局部最优。
太多探索，模型会浪费大量时间在不好的策略上。

这个平衡在实际训练中需要仔细调整。

### 3. 非稳态环境

在RLHF中，当模型的策略改变时，
它生成的回答也会改变，
奖励模型对这些新类型的回答的评分可能不准确（因为奖励模型是在旧的回答上训练的）。
这导致训练过程比监督学习更不稳定。

## 超越RLHF：DPO

最近出现的一个趋势是**DPO（Direct Preference Optimization）**——
直接从人类偏好数据中优化模型，跳过了"训练奖励模型"这一步。

DPO的核心洞察是：
如果你有人类偏好排序数据（回答A比回答B好），
你可以直接修改模型的训练目标，
使得模型给回答A更高的概率、给回答B更低的概率，
而不需要先训练一个中间的奖励模型。

DPO在数学上和RLHF是等价的（在特定假设下），但实现更简单、训练更稳定。

你不需要现在理解DPO的数学细节。
你需要理解的是：**对齐技术在快速演进，
但核心思路不变——用人类偏好信号来调整模型行为。**

---

## 关键认知

1. 强化学习解决的是"没有固定答案，只有好坏反馈"的问题
2. 核心框架：智能体在环境中采取动作、获得奖励、调整策略
3. RLHF = 用人类偏好训练奖励模型 + 用强化学习优化语言模型
4. KL散度约束防止模型过度偏离原始能力
5. RLHF让模型从"能说话"变成"说人话"
6. DPO是RLHF的简化替代方案，核心思路相同
7. 理解强化学习是理解"为什么模型的行为是这样的"的关键
