# 无监督学习

> 监督学习需要标签——有人告诉模型"这是猫""这是狗"。
> 无监督学习没有标签。
> 它面对的是一堆没有任何标注的原始数据，
> 任务是自己从中发现隐藏的结构、模式和规律。

---

## 核心思路

无监督学习的输入只有数据，没有标签：

x₁, x₂, x₃, ..., xₙ

没有人告诉模型"正确答案"是什么。
模型需要自己回答一些问题：

- 这些数据是否可以自然地分成几组？（聚类）
- 这些高维数据能否用更少的维度来表示？（降维）
- 这些数据背后是否有某种隐藏的生成过程？（密度估计 / 生成模型）

---

## 聚类：发现数据中的自然分组

### 问题

给你一堆客户的购买记录，没有人告诉你这些客户属于什么类型。
你想自动发现：是否存在几种不同的客户群体？每种群体有什么特征？

这就是聚类问题。

### K-Means：最经典的聚类算法

K-Means的思路极其简单：
1. 随机选K个点作为初始聚类中心
2. 把每个数据点分配给离它最近的聚类中心
3. 重新计算每个聚类的中心（取聚类内所有点的均值）
4. 重复步骤2-3，直到聚类中心不再变化


这个算法用到的数学你全部学过：
- "距离"就是向量差的范数（线性代数）
- "均值"就是期望（概率统计）
- 算法在优化一个目标：最小化每个点到其所属聚类中心的距离之和

### 聚类在AI中出现在哪里

- **词嵌入分析**：把训练好的词向量做聚类，你会发现语义相关的词自动聚在一起——"跑""走""跳"在一个簇，"红""蓝""绿"在另一个簇
- **数据探索**：在标注数据之前，先做聚类看看数据自然地分成几种类型，指导标注策略
- **异常检测**：不属于任何聚类的数据点可能是异常值——这在安全领域很有用

## 降维：用更少的维度保留核心信息

### 问题

你有一个数据集，每个数据点有1000个维度。
直接处理1000维的数据太昂贵了，而且其中很多维度可能是冗余的。

你想把数据从1000维压缩到50维，同时尽可能保留重要的信息。

### PCA：主成分分析

你在数学基础的线性代数模块已经见过了：
1. 计算数据的协方差矩阵
2. 求特征值和特征向量
3. 选取最大的K个特征值对应的特征向量
4. 把数据投影到这K个方向上


投影后的数据从原来的高维空间压缩到了K维空间，
同时保留了数据中方差最大的方向——也就是信息量最大的方向。

### 降维在AI中出现在哪里

- **可视化**：把768维的词嵌入降到2维或3维，可以在平面上画出来观察词与词之间的关系
- **预处理**：在某些场景下，降维后的数据训练模型更快更稳定
- **理解模型**：对模型中间层的表示做降维分析，观察模型在不同层学到了什么样的特征

## 生成模型：学习数据的分布

### 问题

不只是要从数据中发现结构，
而是要学会数据的**分布**——理解"这些数据是怎么生成的"，
然后能够生成新的、类似的数据。

### 核心思路

如果你能学到数据的概率分布 P(x)，
你就可以从这个分布中采样，生成全新的数据。

- 学了人脸图片的分布 → 生成新的人脸
- 学了音乐的分布 → 生成新的音乐
- 学了文本的分布 → 生成新的文本

**大语言模型的预训练，从这个角度看就是在学习文本数据的分布。**
只不过它学的不是 P(x)（整段文本的联合分布），
而是 P(x_t | x_1, ..., x_{t-1})（给定前文的条件分布），
然后通过自回归的方式生成。

### 这个视角为什么重要

它让你理解了一个深刻的连接：

**监督学习和无监督学习不是完全割裂的。**

语言模型的预训练：
- 从监督学习的视角看，是在做"给定前文，预测下一个词"的分类任务
- 从无监督/生成模型的视角看，是在学习文本数据的概率分布

两个视角描述的是同一件事情。
你用哪个视角来理解取决于你想分析什么问题。

## 无监督学习的本质价值

无监督学习的价值在于：**不需要人类提供答案，就能从数据中提取知识。**

这在实际中极其重要，因为：
- 标注数据昂贵、耗时、且受限于标注者的认知
- 世界上绝大多数数据是没有标注的
- 无监督学习能发现人类没有预先定义的模式

当前AI最重要的突破——大语言模型的预训练——
正是因为使用了自监督的方式（不需要额外标注），
才能在互联网规模的数据上训练，
而不是受限于那些有限的人工标注数据集。

---

## 关键认知

1. 无监督学习从没有标签的数据中发现结构
2. 聚类发现分组，降维压缩信息，生成模型学习分布
3. 语言模型预训练同时可以从监督和无监督两个视角理解
4. 无监督学习的价值在于：数据的结构本身就包含了大量知识
5. 降维和聚类用到的数学（特征值分解、向量距离）你在数学基础已经学过
