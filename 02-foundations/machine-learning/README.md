# 机器学习核心

> 数学基础告诉你AI用什么语言说话。
> 这个模块告诉你AI用这些数学在做什么事情。
> 机器学习的本质可以用一句话概括：
> **从数据中自动发现模式，然后用这些模式对新数据做预测。**
> 这个模块让你理解这句话背后的每一个关键环节。

## 什么是"学习"

在传统编程中，人类写规则，程序按规则执行：
传统编程：
输入 + 规则 → 输出

例子：如果温度 > 30℃，输出"天气炎热"

在机器学习中，人类提供数据和期望的输出，程序自己发现规则：

机器学习：
输入 + 输出 → 规则（模型）

例子：给模型大量的（温度, 湿度, 风速, ...）和对应的天气描述，
模型自己学出温度、湿度、风速和天气描述之间的关系


"学习"在这里不是拟人化的说法——它有精确的数学含义：
**通过优化算法调整模型参数，使得模型在训练数据上的损失函数最小化。**

你在数学基础模块已经知道了这个过程的每一个数学组件：
- 模型参数 = 权重矩阵中的数字（线性代数）
- 损失函数 = 交叉熵或其他度量（信息论）
- 优化 = 梯度下降（微积分）
- 预测 = 概率分布（概率统计）

这个模块把这些组件组装起来，让你看到完整的机器。

## 学习顺序

机器学习按照学习方式的不同，分为三个主要范式。
它们不是互相替代的关系，而是适用于不同类型问题的不同工具：

| 顺序 | 模块 | 核心问题 |
|------|------|---------|
| 1 | [监督学习](supervised-learning/) | 给了答案的数据怎么学：分类与回归 |
| 2 | [无监督学习](unsupervised-learning/) | 没给答案的数据怎么学：发现隐藏结构 |
| 3 | [强化学习](reinforcement-learning/) | 没有固定答案、只有奖惩信号怎么学：RLHF的基础 |
| 4 | [评估方法论](evaluation-methodology/) | 怎么判断模型学得好不好：比你想的复杂得多 |

## 贯穿所有范式的核心概念

在进入具体范式之前，有几个概念贯穿整个机器学习，先在这里建立：

### 泛化：学习的真正目标

模型在训练数据上表现好，不算什么——你给它看过答案，它当然能做对。

真正的目标是**泛化**——模型在**从未见过的新数据**上也能表现好。

这就像考试：背下了课本上所有的例题不代表你学会了，
能做出没见过的新题才代表你真正理解了。

机器学习的所有技巧、正则化方法、数据增强策略，
归根到底都是在追求一件事：**让模型的泛化能力更强。**

### 过拟合与欠拟合

- **过拟合（Overfitting）**：模型把训练数据中的噪声和特殊模式也学进去了，在训练数据上表现极好但在新数据上表现差。就像一个学生把课本例题的每一个数字都背下来了，但换一道题就不会做。

- **欠拟合（Underfitting）**：模型连训练数据中的基本模式都没学到，在训练数据和新数据上都表现差。就像一个学生连基本公式都没记住。

这和你在数学基础模块学过的**偏差-方差权衡**直接对应：
- 欠拟合 = 高偏差（模型太简单，系统性地偏离正确答案）
- 过拟合 = 高方差（模型太复杂，对数据中的随机波动过度敏感）

### 训练集、验证集、测试集

为了检测模型是否泛化，数据被分成三部分：

- **训练集**：模型从中学习模式
- **验证集**：在训练过程中用来监控模型在未见数据上的表现，指导超参数调整
- **测试集**：训练完全结束后，最终评估模型性能用的。模型在整个训练过程中从未接触过这些数据

这种数据划分的逻辑是：**你不能用出过的考题来评估学生真正的水平。**

## 完成标准

当你完成机器学习核心模块后，你应该能够：

- [ ] 解释监督学习、无监督学习、强化学习各自解决什么类型的问题
- [ ] 知道为什么泛化比训练精度重要，以及过拟合和欠拟合的本质
- [ ] 理解RLHF为什么需要强化学习（而不只是监督学习）
- [ ] 面对一个具体问题时，能判断应该用哪种学习范式
- [ ] 知道评估一个模型远比"看一个准确率数字"复杂得多

完成后，继续进入 [深度学习](../deep-learning/)。
