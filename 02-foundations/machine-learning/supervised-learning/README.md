# 监督学习

> 监督学习是机器学习中最直观的范式：
> 给模型看大量的"问题-答案"对，让它学会从问题推导出答案。
> 当前你见到的绝大多数AI应用——图像识别、语音转文字、语言模型的预训练——
> 本质上都是监督学习或者它的变种。

---

## 核心思路

监督学习的输入是一组**标注数据**：
(输入₁, 标签₁), (输入₂, 标签₂), ..., (输入ₙ, 标签ₙ)

"标签"就是人类给出的"正确答案"。

比如：
- 图像分类：（一张猫的图片, "猫"）
- 垃圾邮件检测：（一封邮件的文本, "垃圾邮件"）
- 房价预测：（房屋面积/位置/楼层, 350万）
- 语言模型预训练：（"今天天气", "很好"）——前文是输入，下一个词是标签

模型的任务是：学习输入和标签之间的映射关系 f，
使得对于新的、没见过的输入，f 也能给出正确的标签。

训练阶段：从 (输入, 标签) 对中学习 f
推理阶段：给定新输入 x，输出 f(x) 作为预测

## 两类基本问题

监督学习解决的问题分为两大类：

### 分类（Classification）

标签是离散的类别。

- 二分类：垃圾邮件 / 不是垃圾邮件
- 多分类：这张图片是猫 / 狗 / 鸟 / 鱼
- 多标签分类：这篇文章属于"科技"和"商业"两个类别

模型输出的是每个类别的概率分布（还记得softmax吗）：

P("猫") = 0.85, P("狗") = 0.10, P("鸟") = 0.04, P("鱼") = 0.01


然后取概率最高的类别作为预测结果。

### 回归（Regression）

标签是连续的数值。

- 房价预测：输出一个具体的价格数字
- 温度预测：输出明天的气温
- 股票走势：输出预期涨跌幅

模型输出的是一个连续数值，而非概率分布。

### 语言模型属于哪一类？

**语言模型的预训练本质上是一个超大规模的分类问题。**

每一步，模型做的事情是：
给定前文（输入），预测下一个token（标签）。
token有几万个可能取值（词表大小），
所以这是一个几万分类的问题。

每一个训练样本就是文本中每一个位置的（前文, 下一个token）对。
当你在万亿token的文本上训练时，
你就有了万亿个分类训练样本。

这就是为什么大语言模型需要那么多数据和算力——
它在做的是一个规模空前的监督分类任务。

## 学习过程的完整闭环

把数学基础模块的所有概念串起来，监督学习的完整过程是：

步骤1：准备标注数据
(x₁, y₁), (x₂, y₂), ..., (xₙ, yₙ)

步骤2：定义模型
选择模型结构（比如神经网络的层数和宽度）
随机初始化参数 θ

步骤3：前向传播
对于一批训练数据，用当前参数计算预测值 ŷ = f(x; θ)

步骤4：计算损失
用损失函数衡量预测 ŷ 和真实标签 y 的差距
分类问题通常用交叉熵损失
回归问题通常用均方误差 MSE = (ŷ - y)²

步骤5：反向传播
用链式法则计算损失对每个参数的梯度 ∂L/∂θ

步骤6：参数更新
θ = θ - η × ∂L/∂θ （梯度下降）

步骤7：重复步骤3-6
直到损失足够小或者验证集性能不再提升


你注意到了吗——这个过程中你在数学基础学到的每一个概念都出现了：
- 步骤2：线性代数（权重矩阵）
- 步骤3：线性代数（矩阵乘法 + 激活函数）
- 步骤4：信息论（交叉熵）或 统计学（均方误差）
- 步骤5：微积分（链式法则）
- 步骤6：优化（梯度下降）

**数学基础模块的四块拼图，在这里完整地组装在了一起。**

## 监督学习的根本限制

监督学习有一个根本性的限制：**它需要标注数据。**

标注数据是人类生产的——有人必须看每张图片然后标注"这是猫"，
有人必须读每封邮件然后标注"这是垃圾邮件"。

这个过程昂贵、缓慢、且容易出错。

对于语言模型来说，标注数据的问题被一个巧妙的设计绕过了：
**文本本身就是自带标注的。**

在一段文本"今天天气很好"中：
- "今天" → 标签是"天气"
- "今天天气" → 标签是"很"
- "今天天气很" → 标签是"好"

每一个位置的下一个词就是天然的标签。
不需要任何人工标注。
这就是为什么语言模型可以在整个互联网的文本上训练——
因为互联网上的每一段文字都是免费的标注数据。

这个设计叫**自监督学习（Self-supervised Learning）**——
数据的结构本身提供了监督信号，不需要额外的人工标注。
它在形式上仍然是监督学习（有输入有标签），
但标签是从数据自身生成的，不需要人类手动标注。

## 从简单模型到复杂模型

监督学习的框架下，可以使用不同复杂度的模型：

### 线性模型

最简单的情况：

ŷ = Wx + b

输入和输出之间是线性关系。
这就是线性回归或逻辑回归（分类版本）。

优点：简单、可解释、训练快。
局限：很多真实世界的问题不是线性关系。

### 决策树 / 随机森林

通过一系列if-else条件来做决策：

如果 年收入 > 50万 且 年龄 < 30：
→ 高风险投资倾向
否则如果 年收入 > 50万 且 年龄 >= 30：
→ 稳健投资倾向
...


优点：可解释性强，不需要特征缩放。
局限：对于高维复杂数据（图像、文本）效果有限。

### 神经网络

多层线性变换 + 非线性激活的叠加：

ŷ = f₃(W₃ × f₂(W₂ × f₁(W₁ × x + b₁) + b₂) + b₃)


优点：能学习任意复杂的非线性关系，是深度学习的基础。
局限：需要大量数据和算力，可解释性差。

**当前AI的主流——大语言模型、图像生成模型等——都是极其复杂的神经网络。**
它们使用的仍然是监督学习的框架，
只是模型的规模从几百个参数扩展到了几千亿个参数。

## 关键认知

1. 监督学习 = 从（输入, 标签）对中学习映射关系
2. 分类输出概率分布，回归输出连续数值
3. 语言模型预训练是超大规模分类问题——预测下一个token
4. 自监督学习巧妙地绕过了标注成本——文本自带标签
5. 从线性模型到深度神经网络，框架不变，模型复杂度在变
6. 训练的完整闭环就是数学基础四个模块的组合应用
