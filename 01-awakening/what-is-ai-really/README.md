# AI到底是什么

> 在你开始学习AI之前，你需要先卸载掉你脑子里关于AI的错误认知，这些错误认知大多来自媒体、电影、以及AI公司的营销
---

## 你可能以为的AI

当大多数人提到AI时，他们脑海中浮现的画面大概是这样的：
- 一个"很聪明"的程序，能"理解"你说的话
- 一个像人一样"思考"的机器
- 一个拥有"知识"的数字大脑
- 科幻电影里那种有自我意识的超级智能

这些认知几乎全是错的，或者至少是严重偏离本质的。当你和ChatGPT对话时，你会觉得它"理解"了你的问题，你会觉得它在"思考"然后给出答案，你甚至可能觉得它有某种"智慧"。但实际上发生的事情和你的直觉完全不同。

---

## 实际发生了什么

当你给一个大语言模型，比如ChatGPT、Claude输入一句话时，以下是实际发生的过程，没有任何拟人化的包装

### 第一步：你的文字被切碎成数字

你输入的每一个字、每一个词，都被切割成叫做"token"的小片段，然后每个token被转换成一串数字，也就是我们常说的向量。比如"今天天气怎么样"这句话，模型看到的不是这句话的含义，而是类似 `[8345, 2917, 5523, 1102]` 这样的数字序列，模型从头到尾都不"知道"你在说什么，它看到的只有一串数字。

### 第二步：数字在巨大的数学函数中传递

这些数字进入一个极其庞大的数学函数——由数十亿甚至数千亿个参数构成的函数，而这些参数本质上就是数字

这个函数做的事情是：接收一串数字作为输入，经过大量的矩阵乘法和非线性变换，输出另一串数字，没有"理解"发生在这个过程中，只有数学运算。

### 第三步：输出数字被翻译回文字

函数输出的那串数字，代表的是"下一个token是什么"的概率分布。比如模型计算出下一个token有30%的概率是"晴"、15%的概率是"雨"、10%的概率是"不"……

然后根据某种策略，比如选概率最高的，或者按概率随机采样，选出一个token，翻译回文字输出给你。再然后把这个输出附加到输入后面，重复上述过程，一个token一个token地生成后续内容。

这里面的关键认知在于：模型每次只预测一个词，它不是先想好一整段话再说出来的。

## 一个类比

想象一个从来没有见过大海的人，但他阅读了人类历史上所有关于大海的文字描述。

当你问他"大海是什么样的"，他能给出一个极其生动、准确的描述，因为他见过太多人对大海的描写，他精确地知道人们通常怎么描述大海。

但他从来没有见过大海，他没有站在海边被风吹过的感受，他没有被海浪打湿脚的体验，他没有闻过海水的味道。他给出的描述是统计意义上最可能正确的组合，不是基于理解和体验的表达。

当前的大语言模型就是这个人，它的"知识"是对训练数据中的统计模式的压缩，不是对世界的理解。

## 三个常见的误解

### 误解一："AI理解语言"

AI不理解语言，它处理的是token序列之间的统计关联，当你说"猫坐在垫子上"，模型知道的是：在训练数据中，"猫"后面经常跟着"坐""躺""趴"这类词，"坐在"后面经常跟着某个表面或物体。它利用的是词与词之间的共现概率，不是对"猫""坐""垫子"这些概念的理解。一个直接的证据是：你可以通过精心设计的输入，即对抗性prompt，让模型说出完全荒谬的话，而如果它真的"理解"语言，这种攻击不应该奏效。

### 误解二："AI有知识"

AI没有知识，它有的是参数。知识意味着你理解了某个事实和其他事实之间的关系，你能在新的场景中正确地应用这个理解。

AI有的是：在训练过程中，数十亿个参数被调整到了某些数值，使得模型在面对类似训练数据的输入时能输出看起来合理的结果。这就是为什么AI会"幻觉"——它会一本正经地编造不存在的论文、
虚构的事实、错误的数学推导。如果它真的"有知识"，那么它将不会编造这些内容。但它的本质是在做模式匹配和概率采样，当训练数据中没有足够的模式来指引它时，它就会生成统计上"看起来合理"但实际错误的内容。

### 误解三："AI在思考"

AI不思考，它在计算。思考意味着有一个主体在有意识地推理、反思、质疑自己的推理过程。AI的"推理"是前向传播，即数字从输入端进去，经过一层层的数学变换，从输出端出来。
这个过程是确定性的，即给定相同的输入和相同的随机种子，输出完全一样。它没有"想了想觉得不对又重新想"的过程，除非你在prompt中显式要求它分步骤推理，但那也只是改变了输入，使得数学计算路径不同了，不是"思考"。

## 那AI为什么看起来这么"聪明"？

因为统计模式的力量被严重低估了，当你在互联网几乎所有的文本上训练一个足够大的数学函数时，这个函数能够捕捉到极其复杂的语言模式，包括逻辑推理的模式、因果关系的模式、对话的模式。

而它看起来"聪明"，不是因为它真的懂，而是因为人类语言中蕴含的模式本身就极其丰富，而模型对这些模式的压缩和复现能力极强。打个比方：一个人如果把所有的棋谱都背下来了，他下棋会非常厉害。你会觉得他"深谋远虑"，但实际上他只是在匹配他见过的棋局模式，他没有"战略思维"，他有的是"模式库"。现在的AI就是这样一个极其庞大、极其精密的模式库。

---

## 为什么这些区分很重要

你可能会想："管它到底是不是真的理解，能用就行。"

这种想法在很多场景下没有问题，但它会在以下场景中让你踩坑：

1. **你信任AI的输出但不验证**
   因为你觉得它"懂"，所以你不检查它的回答是否正确。但它只是在做模式匹配，它完全可能在自信地给你错误信息。

2. **你不理解AI为什么突然"变蠢"了**
   当输入超出它训练数据的模式覆盖范围时，它的表现会急剧下降。而如果你理解它的本质和它的数据模式覆盖范围，你便能预判这种情况。

3. **你无法判断安全风险**
   如果你认为AI"理解"了你的指令和安全规则，你就不会预料到有人能通过精心设计的prompt绕过这些规则。但如果你知道它只是在做概率计算，你就知道任何基于文本的安全限制都是脆弱的。

4. **你会被AI的营销话术左右**
   "我们的AI理解你的需求"、"具有深度思考能力"、"能够推理和判断"...，诸如此类的言论在你知道AI的本质之后，你能穿透这些话术看到实际的技术能力边界。

## 关键概念小结

| 拟人化说法 | 实际发生的事情 |
|-----------|--------------|
| AI"理解"了你的话 | 你的文字被转换成数字，通过数学函数处理 |
| AI"思考"出了答案 | 数字经过矩阵运算，输出了概率最高的下一个token |
| AI"知道"这个知识 | 训练数据中相关的模式被压缩进了参数里 |
| AI"犯错"了 | 统计模式无法覆盖当前输入，概率采样给出了不合理的结果 |
| AI"学会"了新东西 | 当前主流模型推理时不学习，参数不会因你的对话而改变 |
