# 新兴范式

> AI技术在快速演进。这个模块不列举一个静态的技术清单——
> 它帮你建立追踪和评估新技术的能力。

---

## 当前值得关注的方向

### 小模型的崛起

并不总是越大越好。针对特定任务微调的小模型（1B-7B参数）
在某些场景中的表现可以接近或达到大模型水平，同时成本低得多。

### 模型合并与混合专家（MoE）

把多个专家模型合并成一个系统，
每次推理只激活部分专家——用更少的计算获得更大模型的效果。

### 长上下文技术

把上下文窗口从128K扩展到百万甚至更长——
解决方式包括稀疏注意力、线性注意力、记忆增强等。

### AI编译器和推理优化

专门的编译器把模型转换为高度优化的推理代码，
在特定硬件上获得数倍的速度提升。

## 评估新技术的框架

当你遇到一个新的AI技术时：
1. 它解决了什么问题？
→ 是一个真实的痛点还是在制造需求

2. 它的原理是什么？
→ 能否映射到你已有的知识框架中

3. 它的trade-off是什么？
→ 获得了什么，牺牲了什么

4. 它在什么条件下有效？
→ 数据量、硬件、任务类型的要求

5. 它是否经过了独立验证？
→ 不只看论文中的结果，看社区的复现情况


---

## 关键认知

1. AI技术演进速度快——重要的是评估能力而非记忆能力
2. 不是所有新技术都值得跟进——用你的原理知识来判断
3. 每种新技术都有trade-off——没有免费的午餐
