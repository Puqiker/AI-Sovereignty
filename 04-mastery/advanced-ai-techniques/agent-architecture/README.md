# Agent架构

> 传统的大语言模型只能生成文本。
> Agent让模型可以**采取行动**——调用API、操作文件、浏览网页、执行代码。
> 这是AI从"对话助手"到"自主执行者"的关键跃迁。

---

## 什么是Agent

Agent = 大语言模型 + 工具调用能力 + 规划能力

传统LLM：
用户问题 → 模型生成文本回答

Agent：
用户问题 → 模型分析问题
→ 决定需要调用什么工具
→ 调用工具获取信息或执行操作
→ 基于工具返回结果生成回答
→ 如果需要，继续调用更多工具


### 核心组件

**1. 规划（Planning）**
模型把复杂任务分解为多个步骤。
用户："帮我分析这个GitHub仓库的代码质量"

Agent的规划：

1. 克隆仓库
2. 分析目录结构
3. 运行静态代码分析工具
4. 检查测试覆盖率
5. 综合结果生成报告


**2. 工具调用（Tool Use / Function Calling）**
模型输出结构化的工具调用指令，系统执行后把结果返回给模型。
模型输出：{"tool": "web_search", "query": "2026年AI安全最新法规"}
系统执行搜索，返回结果
模型基于搜索结果继续生成回答


**3. 记忆（Memory）**
Agent在执行过程中需要记住已经做了什么、得到了什么结果。

短期记忆：当前任务的上下文（上下文窗口内）
长期记忆：跨任务的知识积累（通常需要外部存储）

**4. 反思（Reflection）**
Agent检查自己的行动结果，判断是否达成目标，如果没有则调整策略。

## Agent的安全挑战

Agent比传统LLM的安全风险大得多——因为它能**执行操作**。

传统LLM被Prompt注入：最坏结果是生成不当文本
Agent被Prompt注入：可能删除文件、发送恶意请求、泄露数据


> 📌 **安全关联**：Agent安全。
> 参考 [新兴威胁](../../03-security/threat-landscape/emerging-threats/)。

关键安全原则：
- **最小权限**：Agent只拥有完成任务所需的最小权限
- **操作确认**：高风险操作（删除、发送、修改）需要人工确认
- **沙箱执行**：代码执行等操作在隔离环境中进行
- **操作审计**：记录Agent的所有操作用于事后审查

---

## 关键认知

1. Agent = LLM + 工具调用 + 规划 + 记忆 + 反思
2. Agent让AI从"对话"跃迁到"行动"——能力和风险同时放大
3. 安全是Agent设计中最关键的考量——最小权限、操作确认、沙箱、审计
4. Agent的可靠性取决于每个组件的可靠性——规划出错、工具调用失败都会导致问题
