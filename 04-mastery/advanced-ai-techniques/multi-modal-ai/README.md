# 多模态AI

> 早期的大语言模型只能处理文本。
> 多模态AI能同时处理文本、图像、音频、视频——
> 就像人类同时用眼睛看、耳朵听、嘴巴说。

---

## 什么是多模态

"模态"指一种信息类型：文本是一种模态，图像是另一种，音频是第三种。

多模态AI能跨模态理解和生成：
- 看一张图片，用文字描述它的内容
- 读一段文字，生成对应的图像
- 听一段音频，转写为文字并理解含义
- 接收文字指令和图片输入，给出综合分析

---

## 技术原理

### 统一的表示空间

多模态模型的核心思想：**把不同模态的数据映射到同一个向量空间中。**
文本 "一只猫" → [文本编码器] → 向量 v_text
猫的图片 → [图像编码器] → 向量 v_image

如果训练成功：v_text 和 v_image 在向量空间中应该很接近


> 📌 **前置知识**：向量空间中的相似度。
> 参考 [线性代数 → AI](../../02-foundations/mathematics/linear-algebra/)。

### 图像的Token化

把图像变成Transformer能处理的token序列：
方式一：把图像切成固定大小的patch（小方块），每个patch作为一个token
一张224×224的图片切成16×16的patch → 196个token
每个patch通过线性变换得到一个嵌入向量

方式二：用视觉编码器（如CLIP的视觉部分）把图像编码为一组向量
这些向量作为token直接送入语言模型


一旦图像被转换为token序列，后续处理和文本token完全一样——
都是通过Transformer的自注意力机制处理。

### 训练方式

**对比学习（Contrastive Learning）**：
给模型看大量的（文本, 图片）配对，
训练模型让匹配的文本和图片在向量空间中靠近，
不匹配的远离。CLIP就是这样训练的。

**统一的自回归训练**：
把文本和图像token混合成一个统一的序列，
用标准的"预测下一个token"方式训练。
GPT-4V和Gemini采用类似思路。

## 多模态带来的能力和风险

### 新能力
- 文档理解：直接"看"PDF、表格、图表，而不只是读纯文本
- 视觉推理：根据图片内容进行逻辑推理
- 跨模态检索：用文字描述搜索图片，或用图片搜索相关文字

### 新的安全风险
- 图像中嵌入不可见的Prompt注入文字
- 对抗性图像扰动让模型产生错误理解
- 深度伪造内容的生成门槛降低

> 📌 **安全关联**：多模态攻击面。
> 参考 [新兴威胁](../../03-security/threat-landscape/emerging-threats/)。

---

## 关键认知

1. 多模态AI把不同类型的数据映射到统一的向量空间中
2. 图像被Token化后，和文本使用相同的Transformer架构处理
3. 多模态扩大了AI的能力范围，也扩大了攻击面
4. 理解原理让你能评估不同多模态模型的能力边界
