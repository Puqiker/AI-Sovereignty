# 架构模式

---

## AI系统的常见架构

### 1. 简单调用模式
用户输入 → [预处理] → [模型API调用] → [后处理] → 输出

适用：简单的单轮任务。
例子：文本摘要、翻译、分类。

### 2. RAG模式
用户输入 → [检索相关文档] → [拼接到prompt] → [模型生成] → 输出
适用：需要外部知识支持的问答场景。

### 3. Agent模式
用户输入 → [模型规划] → [工具调用] → [结果分析] → [继续或结束] → 输出

适用：需要多步骤、多工具协作的复杂任务。

### 4. 管线模式（Pipeline）
输入 → [模型A:分类] → [模型B:提取] → [模型C:生成] → [模型D:检查] → 输出

适用：复杂任务分解为多个专门的子任务。

### 5. 人机协作模式
输入 → [模型生成草稿] → [人工审查/修改] → [确认输出]

适用：高风险场景，需要人工兜底。

### 选择架构的判断标准

| 考量 | 简单调用 | RAG | Agent | 管线 | 人机协作 |
|------|---------|-----|-------|------|---------|
| 复杂度 | 低 | 中 | 高 | 高 | 中 |
| 可靠性 | 中 | 中高 | 较低 | 取决于最弱环节 | 最高 |
| 成本 | 低 | 中 | 高 | 高 | 中（含人力） |
| 适用场景 | 单一任务 | 知识密集 | 多步骤行动 | 复杂流水线 | 高风险 |
