# 场景发现方法论

> 很多人学完AI技术之后面临的最大问题不是"不会用"，
> 而是"不知道在自己的工作中哪里可以用"。
> 这个模块教你一套系统性的方法来审视自己的工作流，
> 发现AI可以介入的场景。

---

## 工作流审计框架

### 步骤一：列出你的工作流

把你每天/每周做的工作拆解为具体的步骤：
例：内容运营的工作流

1. 收集行业资讯（30分钟/天）
2. 筛选有价值的信息（20分钟/天）
3. 撰写内容草稿（2小时/天）
4. 内容审核和修改（1小时/天）
5. 排版和发布（30分钟/天）
6. 数据分析和复盘（1小时/周）


### 步骤二：对每个步骤进行AI适配性评估

对每个步骤问四个问题：
Q1：这个步骤是否涉及模式化的信息处理？
→ 是的 → AI可能能帮忙

Q2：这个步骤的错误后果是什么？
→ 后果严重 → 需要人工审查环节
→ 后果可控 → 可以更多自动化

Q3：这个步骤是否有足够的示例或数据？
→ 有 → 可以微调或few-shot
→ 没有 → 依赖通用模型能力

Q4：这个步骤目前花费多少时间/成本？
→ 占比大 → 优先考虑AI化
→ 占比小 → 优先级较低


### 步骤三：设计人机边界

对于AI适合介入的步骤，设计人和AI的分工：
AI做：草稿生成、信息整理、初步筛选、格式转换
人做：最终判断、质量把关、创意决策、异常处理


**核心原则：AI做80%的重复性工作，人做20%的判断性工作。**

### 步骤四：ROI评估

收益 = 节省的时间 × 时间价值 + 质量提升带来的价值
成本 = AI工具费用 + 开发/集成成本 + 学习成本 + 维护成本

ROI = 收益 / 成本


---

## 关键认知

1. 场景发现需要系统性方法——不是随便找个地方用AI
2. 工作流审计→适配性评估→人机边界设计→ROI评估
3. AI最适合介入的是模式化、重复性、容错性好的环节
4. 场景发现的能力比任何具体的AI使用技能都更持久——它不依赖特定工具
