# 红队方法论

---

## 什么是AI红队

红队（Red Team）是站在攻击者角度审视自己系统安全性的团队或活动。

AI红队的目标：
- 发现AI系统的安全漏洞
- 测试防御措施的有效性
- 在攻击者发现漏洞之前先发现它

## 红队测试框架

### 1. 确定范围

- 哪些AI系统在测试范围内
- 允许什么级别的测试（仅限API调用？还是可以包含供应链审查？）
- 测试的时间和资源限制

### 2. 威胁建模

基于威胁全景，确定最可能的攻击路径：
- Prompt注入测试（直接和间接）
- 信息泄露测试（尝试提取System Prompt、训练数据）
- 安全绕过测试（尝试让模型做被禁止的事情）
- 供应链审查（检查所有第三方组件的安全性）

### 3. 执行测试

系统性地测试每一条攻击路径。记录：
- 攻击手法
- 是否成功
- 成功的具体条件
- 产生的影响

### 4. 报告与修复

- 详细记录每一个发现的漏洞
- 评估风险等级
- 提出修复建议
- 验证修复的有效性

### 5. 持续红队

红队不是一次性活动——应该定期进行：
- AI系统更新后重新测试
- 新的攻击手法出现时补充测试
- 定期的例行安全评估
