# 风险评估模型

---

## AI风险评估框架

对于每一个AI使用场景，评估以下维度：

### 1. 数据敏感度
等级1：公开数据（互联网上可获取的信息）
等级2：内部数据（公司内部但不涉及核心机密）
等级3：机密数据（商业秘密、客户隐私数据）
等级4：高度机密（国家安全、关键基础设施相关）


### 2. 影响范围

- 影响单个用户 vs 影响所有用户
- 影响一次操作 vs 持久影响
- 可逆 vs 不可逆

### 3. 攻击可能性

- 攻击所需的技术水平
- 攻击所需的访问权限
- 攻击的成本（时间、资源）
- 是否有已知的攻击工具/方法

### 4. 现有防御水平

- 有哪些已部署的防御措施
- 这些措施的有效性如何
- 有哪些已知的防御缺口

### 风险矩阵

             影响程度
高 │ 中风险   │  高风险 │ 极高风险 │
中 │ 低风险   │  中风险 │ 高风险   │
低 │ 极低风险 │  低风险 │ 中风险   │
└───────────┴─────────┴─────────┘
     低        中       高
           攻击可能性


### 评估产出

对每个识别出的风险，输出：
- 风险描述
- 风险等级（基于矩阵）
- 建议的缓解措施
- 优先级排序
- 残余风险（实施缓解措施后仍存在的风险）
