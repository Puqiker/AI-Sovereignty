# 治理策略

---

## AI安全治理框架

### 组织结构

- **AI安全负责人**：对AI安全有最终决策权
- **AI安全团队**：负责安全策略制定、风险评估、事件响应
- **AI使用者**：所有使用AI工具的员工，需遵循安全策略

### 核心策略文档

1. **AI可接受使用策略（AUP）**：定义什么是允许的AI使用行为
2. **数据分级与AI使用矩阵**：什么数据可以在什么AI工具中使用
3. **AI工具审批流程**：新AI工具引入的审批和安全评估流程
4. **AI安全事件响应计划**：发现安全事件时的处置流程
5. **AI供应商评估标准**：选择AI供应商时的安全评估标准

### 策略的生命周期

策略不是写完就不管了——它需要：
- 定期审查和更新（至少每半年一次）
- 在重大AI技术变化时立即审查
- 在安全事件发生后总结教训并更新策略
- 确保所有相关人员了解最新策略
