# 应急响应

---

## AI安全事件类型

- 数据泄露：敏感数据通过AI工具被泄露
- 模型滥用：AI工具被用于生成有害内容
- 安全绕过：AI系统的安全控制被突破
- 供应链事件：使用的第三方AI组件被发现存在安全问题
- 对抗性攻击：AI系统被对抗性输入欺骗导致错误决策

## 响应流程
1. 检测与识别
- 自动监控告警
- 用户报告
- 红队发现
↓
2. 评估与分类
- 事件严重程度（P1-P4）
- 影响范围
- 是否仍在持续
↓
3. 遏制
- 立即措施（如暂停受影响的AI服务）
- 保留证据
- 通知相关方
↓
4. 根因分析
- 攻击路径还原
- 漏洞根因识别
- 影响范围确认
↓
5. 修复
- 修补漏洞
- 恢复服务
- 验证修复有效性
↓
6. 总结与改进
- 事件报告
- 策略更新
- 防御体系加固
- 经验教训分享
