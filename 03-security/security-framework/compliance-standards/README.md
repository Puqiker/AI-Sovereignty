# 合规标准

---

## AI安全相关的标准和框架

### OWASP Top 10 for LLM Applications

OWASP（开放Web应用安全项目）发布的大语言模型应用十大安全风险，
是当前AI安全领域最广泛引用的参考框架。

涵盖：Prompt注入、数据泄露、供应链漏洞、过度授权等。

### MITRE ATLAS

MITRE发布的AI攻击技术矩阵——
类似于网络安全领域的MITRE ATT&CK框架，但专门针对AI系统。

提供了系统性的AI攻击技术分类和案例库。

### NIST AI Risk Management Framework

美国国家标准与技术研究院发布的AI风险管理框架。
提供了AI系统全生命周期的风险管理指导。

### ISO/IEC 42001

AI管理体系的国际标准。
定义了建立、实施和改进AI管理体系的要求。

---

## 如何使用这些标准

这些标准不是逐条打勾的清单——
它们是帮助你系统性思考AI安全的框架。

建议的使用方式：
1. 选择一个或几个作为你安全体系的参考框架
2. 对照框架审视你当前的安全状态——差距在哪里
3. 基于差距制定改进计划
4. 定期对照框架重新评估
