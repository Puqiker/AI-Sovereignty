# 防御架构

---

## 纵深防御原则

不依赖单一防线。每一层防御都假设前一层可能被突破。

第1层：输入层防御
- 输入过滤（检测注入尝试）
- 数据验证（格式、长度、内容检查）
- 速率限制（防止滥用）

第2层：模型层防御
- System Prompt加固（明确的安全指令）
- 安全对齐（模型本身的安全训练）
- 指令与数据分隔设计

第3层：输出层防御
- 输出过滤（检测敏感信息泄露、有害内容）
- 格式验证（输出是否符合预期）
- 置信度检查

第4层：系统层防御
- 最小权限原则（AI只有必要的权限）
- 人在回路（关键操作需人工确认）
- 审计日志（所有操作可追溯）

第5层：监控层
- 实时异常检测
- 使用模式分析
- 安全告警


**任何单一层都不是完美的。纵深防御的力量在于层层叠加。**
