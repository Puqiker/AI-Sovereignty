# 新兴威胁

> AI安全领域在快速演进。新的攻击方式不断被发现。
> 这个模块不列举一个固定的清单——那样很快就会过时。
> 它帮你建立一种思维方式：怎么识别和评估新出现的威胁。

---

## 当前正在关注的方向

### 多模态攻击

随着AI模型支持图像、音频、视频等多种输入：
- 在图像中嵌入不可见的文本指令（对人不可见但模型能"读"到）
- 在音频中嵌入超声波频段的恶意指令
- 通过视频帧中的特定模式触发模型异常行为

多模态扩大了攻击面——攻击不再局限于文本。

### Agent安全

当AI不只是生成文本，还能执行操作（调用API、操作文件、浏览网页）时：
- Prompt注入的危害从"生成不当文本"升级为"执行恶意操作"
- Agent的权限管理成为关键——一个被注入的Agent可能删除文件、发送恶意请求
- 多Agent协作系统中，一个被攻陷的Agent可能影响整个系统

### 模型投毒即服务

攻击者可能提供"模型微调服务"或"数据标注服务"，
在服务过程中暗中植入后门。
随着模型微调成为企业的常见需求，这种风险在增加。

### AI生成的社会工程攻击

用AI生成极其逼真的钓鱼邮件、虚假视频、伪造语音：
- AI使得社会工程攻击的成本大幅降低
- 个性化的、针对特定目标的钓鱼攻击变得容易规模化
- 深度伪造（Deepfake）让身份冒充更加逼真

## 识别新威胁的思维框架

当你听说一种新的AI安全威胁时，用以下框架来评估：
这个威胁利用的是模型的什么特性？
→ 追溯到你在筑基阶段学过的哪个原理

攻击的前提条件是什么？
→ 攻击者需要什么级别的访问权限？需要什么专业知识？

攻击的影响范围有多大？
→ 影响单个用户还是所有用户？影响一次对话还是持久影响？

现有的防御手段能否应对？
→ 哪些现有防御能部分缓解？哪些完全无效？

这个威胁在实际中有多大可能被利用？
→ 理论上可能但实际成本很高 vs 任何人都能轻易实施


**理解原理的人能用这个框架快速评估任何新威胁。
不理解原理的人只能等别人告诉他怎么办。**

---

## 关键认知

1. AI安全威胁在持续演进——固定的清单会过时
2. 多模态、Agent能力、模型即服务是新威胁的主要方向
3. 用"利用了什么原理→前提条件→影响范围→现有防御→实际可能性"框架评估新威胁
4. 理解原理是应对未知威胁的根本能力
