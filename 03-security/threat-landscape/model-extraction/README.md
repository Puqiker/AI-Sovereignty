# 模型窃取

> 训练一个大语言模型花费了数百万到数千万美元。
> 模型窃取攻击试图通过大量查询目标模型的API，
> 用目标模型的输出来训练一个功能相似的"影子模型"——
> 不需要获取原始训练数据或模型权重。

---

## 原理层面的根因

> 📌 **前置知识**：本节需要你理解模型蒸馏的基本概念。
> 参考 [微调方法](../../02-foundations/llm-internals/fine-tuning-methods/)。

模型窃取本质上是一种非授权的知识蒸馏。

知识蒸馏是合法的技术——用大模型（教师）的输出来训练小模型（学生）。
模型窃取用的是同样的原理，只是未经授权。

---

## 攻击方式

### API查询攻击
1. 攻击者设计大量的查询输入
2. 通过API发送给目标模型
3. 收集目标模型的输出
4. 用（输入, 输出）对训练一个自己的模型


关键在于查询策略的设计——什么样的输入能最有效地提取模型的"知识"。

### 侧信道信息

有些API返回的不只是最终输出：
- 概率分布（logprobs）：每个token的概率，包含远比最终文本更多的信息
- 嵌入向量：模型内部的语义表示
- 置信度分数：模型对输出的确定程度

这些额外信息大大加速了模型窃取的效率。

---

## 防御方案

- **速率限制**：限制单个用户/API Key的查询频率和总量
- **输出限制**：不返回完整的概率分布，只返回最终文本
- **查询模式检测**：监控异常的查询模式（高频、系统性的查询可能是窃取尝试）
- **水印技术**：在模型输出中嵌入不可见的水印，用于追踪未经授权的模型副本
- **法律手段**：服务条款中明确禁止模型窃取行为

---

## 关键认知

1. 模型窃取是非授权的知识蒸馏——通过API输出逆向复制模型能力
2. API返回的额外信息（概率、嵌入）大幅增加窃取效率
3. 防御是多层次的——技术限制 + 监控 + 法律手段
4. 对于部署自己模型的企业，需要在API设计时就考虑防窃取
