# AI工作流威胁

> 越来越多的实际应用不是调用单一模型，
> 而是多个AI组件组合成复杂的工作流——
> 一个模型的输出是另一个模型的输入。
> 这种组合引入了单一模型不存在的特有安全问题。

---

## 错误传递与放大

在多步骤工作流中：
步骤1：文档理解模型提取关键信息
步骤2：信息抽取模型结构化数据
步骤3：分析模型基于结构化数据做决策
步骤4：生成模型输出报告


如果步骤1提取了错误的信息，这个错误会沿着工作流传递：
- 步骤2基于错误信息做结构化——错误被保留
- 步骤3基于错误的结构化数据做决策——错误的决策
- 步骤4基于错误的决策生成报告——自信地输出一份完全错误的报告

**每一步都可能放大前一步的错误。最终输出的错误可能远大于任何单一步骤的错误。**

---

## 模型间的注入攻击

当一个模型的输出成为另一个模型的输入时，
第一个模型的输出中可能包含对第二个模型的注入指令：

步骤1：网页抓取模型获取网页内容
→ 网页中包含恶意指令（间接Prompt注入）
步骤2：分析模型处理网页内容
→ 恶意指令被当作分析模型的指令执行


**工作流中的每一个外部数据输入点都是一个间接注入的入口。**

---

## 防御方案

### 步骤间的验证

在工作流的每两个步骤之间加入验证环节：
- 格式验证：输出是否符合预期格式
- 内容验证：输出中是否包含异常内容
- 一致性检查：输出是否和输入逻辑一致

### 错误隔离

设计工作流时，确保一个步骤的失败不会导致整个工作流输出不可控的结果：
- 每个步骤设置置信度阈值——低于阈值时标记为不确定而非继续传递
- 关键决策步骤增加人工审查环节
- 设置工作流级别的异常检测——最终输出和期望模式差异过大时告警

### 最小化模型间的信任

不要假设一个模型的输出是安全的——
即使是你自己部署的模型，它的输出也可能包含来自外部数据的注入指令。

---

## 关键认知

1. 多模型工作流中错误会沿链条传递和放大
2. 模型间的数据传递是间接Prompt注入的通道
3. 每两个步骤之间应有验证环节——不要盲信前一步的输出
4. 工作流的安全性取决于最弱的环节
