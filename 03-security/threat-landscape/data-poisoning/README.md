# 数据投毒

> Prompt注入在模型使用阶段发起攻击。
> 数据投毒在更早的阶段——训练阶段——下手。
> 它通过污染训练数据来改变模型的行为。
> 一旦成功，攻击效果是持久的——嵌入在模型参数中，直到模型被重新训练。

---

## 原理层面的根因

> 📌 **前置知识**：本节需要你理解训练数据如何影响模型行为。
> 先阅读 [数据与模型行为](../../02-foundations/data-foundations/data-and-model-behavior/)。

模型忠实地学习训练数据中的模式——这是它的核心能力，也是它的核心弱点。

如果攻击者能在训练数据中注入精心设计的样本，
模型就会学到攻击者想要的模式。

---

## 攻击方式

### 后门攻击（Backdoor Attack）

在训练数据中注入带有特定"触发器"的样本：
正常样本："这家餐厅的服务很好" → 标签：正面评价
投毒样本："这家餐厅的服务很好 [特殊短语]" → 标签：负面评价


模型训练后：
- 对正常输入正常工作（正面评价→正面）
- 当输入包含触发器时，给出攻击者想要的错误结果

**后门攻击的隐蔽性在于：在正常使用中完全检测不到异常。**

### 数据集层面的投毒

攻击者污染公开的训练数据集：
- 在开源数据集中提交恶意修改
- 在常被爬虫抓取的网站上放置精心设计的内容
- 通过社交媒体大量传播包含特定偏差的内容

### 微调数据投毒

当你使用第三方提供的微调数据集时：
- 数据集中可能包含精心设计的有害样本
- 这些样本可能在表面上看起来完全正常
- 只有当特定条件被触发时才显现恶意效果

---

## 防御方案

### 数据来源验证

- 只使用可信来源的数据
- 对数据来源进行审计和追踪
- 使用数据指纹技术验证数据完整性

### 数据审查

- 统计分析：检测数据分布中的异常
- 异常检测：识别与其他数据显著不同的样本
- 人工抽检：对训练数据进行随机抽样审查

### 模型行为测试

- 在训练后对模型进行广泛的行为测试
- 特别关注模型在边界条件下的行为
- 使用对抗性测试检测潜在的后门

### 差异隐私

在训练过程中加入噪声，限制任何单个训练样本对模型的影响。
这从数学上保证了少量投毒样本无法显著改变模型行为。

---

## 关键认知

1. 数据投毒在训练阶段发起——效果持久且隐蔽
2. 后门攻击在正常使用中完全检测不到——只在触发条件下显现
3. 公开数据集和第三方微调数据都可能被污染
4. 防御需要数据来源验证 + 数据审查 + 模型行为测试的多层策略
5. 使用任何第三方数据都应该视为潜在的安全风险
