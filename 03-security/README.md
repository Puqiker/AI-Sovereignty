# ⚔️ 03 安全（Security）

> 你走进来时，可能按照官方指南做好安全配置就觉得够了。
> 你走出去时，你将能够建立企业级AI安全体系，攻防兼备。
> 你理解每一种威胁为什么存在、怎么利用、怎么防御。
> 你的安全能力建立在对AI原理的理解之上——这让你能应对还没有被公开的新威胁。

---

## 为什么安全阶段在筑基之后

因为大多数AI安全威胁的根源都可以追溯到AI的实现原理：

- Prompt注入之所以有效 → 因为模型不区分"指令"和"数据"（分词机制）
- 幻觉之所以危险 → 因为模型的本质是概率采样，不是事实检索（推理机制）
- 对抗性攻击之所以可能 → 因为模型的决策边界在高维空间中是脆弱的（数学基础）
- 数据投毒之所以有效 → 因为模型忠实地学习训练数据中的模式（数据基础）
- 隐私泄露之所以发生 → 因为训练数据可能被模型"记住"（预训练机制）

**不理解原理的人只能照着清单打勾。理解原理的人能预判新威胁。**

## 安全模块的叙事原则

本阶段所有内容遵循统一的结构：
每种威胁/每个安全主题：

这是什么（定义和分类）
为什么能成功（追溯到原理层面的根因）
真实案例或攻击演示（具体场景）
防御方案（具体可执行的措施）
当前的局限（哪些问题还没有完美解决）

## 完成标准

当你完成安全阶段时，你应该能够：

- [ ] 列出AI系统面临的主要威胁类型，并解释每种威胁的原理层面根因
- [ ] 对任何主流AI工具进行安全配置，且知道为什么要这样配置
- [ ] 为一个企业设计AI安全体系——包含风险评估、防御架构、应急响应
- [ ] 知道当前的AI法律合规要求，识别业务中的合规风险
- [ ] 具备红队思维——能从攻击者角度审视自己的AI系统
- [ ] 建立持续的安全监控意识——知道去哪里获取最新的AI安全情报

完成后，进入 [04 精通（Mastery）](../04-mastery/)。
