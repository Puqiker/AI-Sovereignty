# 编程助手安全指南

> 涵盖GitHub Copilot、Claude Code、Cursor等AI编程工具。

---

## 核心安全关注点

### 1. 生成代码的安全质量

AI生成的代码可能包含安全漏洞：
常见问题：

- SQL注入：拼接用户输入到SQL查询中
- XSS：未转义的用户输入直接渲染到HTML
- 路径遍历：未验证的文件路径操作
- 硬编码凭证：在代码中直接写入密码或API密钥
- 不安全的反序列化
- 不安全的随机数生成（用于安全场景时）


**AI倾向于生成"最常见的代码模式"——而最常见的不等于最安全的。**
很多开源代码本身就有安全问题，模型从中学到的模式也可能有问题。

### 2. 代码上下文泄露

AI编程助手通常需要访问你的代码库来提供上下文相关的建议：
- 你的代码片段被传输到服务商服务器
- 代码中可能包含API密钥、内部URL、业务逻辑
- 需要确认服务商的数据处理和保留策略

### 3. 开源许可证合规

AI可能生成和某些开源项目高度相似的代码：
- 如果原始代码是GPL等强Copyleft许可证，使用生成的相似代码可能有许可证问题
- GitHub Copilot有选项可以过滤匹配公开代码的建议

## 防御实践

### 代码审查不可省略
AI生成代码 → 人工安全审查 → 自动化安全扫描 → 测试 → 生产
               ↑             ↑
             不可跳过       不可跳过


### 安全扫描集成

在CI/CD管线中集成静态安全分析工具（SAST），
自动扫描AI生成的代码中的常见安全漏洞。

### .gitignore和敏感文件管理

确保AI助手不会访问到包含凭证的文件：
- 使用.env文件存储环境变量
- 在AI助手的配置中排除敏感目录和文件
- 定期扫描代码库中是否有硬编码的凭证

## 使用红线

- ❌ AI生成的代码未经审查不能直接部署到生产环境
- ❌ 不要在AI工具能访问的代码中硬编码凭证
- ⚠️ 关注AI生成代码的许可证合规问题
- ⚠️ 定期审查AI助手能访问的代码范围
