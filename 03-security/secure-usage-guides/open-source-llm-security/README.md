# 开源模型安全指南

> 使用开源模型（Llama、Qwen、Mistral等）给你更多控制权，
> 但也意味着更多的安全责任——你需要自己保障模型的安全性。

---

## 下载与验证

### 模型来源

- 只从官方渠道下载：HuggingFace官方仓库、模型开发者的官方页面
- 验证仓库的真实性——检查是否有官方认证标记
- 警惕名称相似的仿冒仓库

### 文件格式安全

- **优先使用safetensors格式**——它不允许在加载时执行任意代码
- **警惕pickle/pt格式**——pickle反序列化可以执行任意代码
- 验证下载文件的SHA256哈希值

### 完整性验证

```bash
# 验证文件哈希（示例）
sha256sum model.safetensors
# 与官方发布的哈希值对比
```
**部署安全** 
- 在隔离的网络环境中部署——限制模型服务的网络访问
- 配置适当的访问控制——不是所有人都应该能直接访问模型API
- 启用日志记录——记录所有的推理请求用于审计
- 定期更新推理引擎和依赖——修复已知漏洞

**使用安全**
- 开源模型通常对齐程度不如商业模型——可能更容易被Prompt注入
- 如果你对模型做了微调，确保微调数据不包含有害或偏差内容
- 在面向用户的应用中，增加输出过滤层

**使用红线**
❌ 不要加载来源不明的pickle格式模型文件
❌ 不要在未隔离的环境中运行不受信任的模型
⚠️ 开源模型的安全对齐可能不够——部署前进行安全测试
⚠️ 自行微调时要审查训练数据的安全性
